D-006 (关键) 拆分为三个子问题:
  • D-006a (Vector-FU 交互): Vector 指令执行延迟 (与 VLEN/SEW 相关)、stall 策略 (同类型 FU 阻塞)、与 Issue 级 busy 信号握手

4.4 锁步比较器接口分歧 双审分歧

D-011 (中) 分歧项:
  • MiniMax 建议: 仅在 W 级比较 (PC + 目标寄存器号 + 写入数据), 实现简单
  • Deepseek 建议: W 级 (寄存器) + M 级 (内存写) 双级比较, 更全面但 Store Buffer 同步复杂
融合建议: M2-S1 设计文档先以 W 级比较为基线设计, 预留 M 级扩展接口。若 M+W 双级比较复杂度过高, 向用户提出简化方案 (仅 W 级)。写入 multicore-lockstep.html §3 接口定义。

5.3 1GHz 时序可行性 Deepseek

TSMC 28HPC 工艺 FO4 延迟约 15~20ps, 8 级流水线每级时序预算 = 1000ps / 8 = 125ps。

  • F2 级 (ICache SRAM 读出) 和 M 级 (DCache SRAM 读出) 是最紧张的级, 预估 90~110ps, 占预算 72~88%
  • I 级旁路 mux 预估 80~100ps, 占预算 64~80%, 需仔细物理设计
  • 整体评估: 1GHz 目标可行但紧凑, 需在 RTL 实现中关注 F2/M 级 SRAM macro 选型和 I 级旁路优化
