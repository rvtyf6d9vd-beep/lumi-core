1. 验证方法论

1.1 验证方法组合

Lumi-Core 采用多层次验证方法组合，确保设计正确性和完备性：

验证方法适用范围工具/平台覆盖目标
Simulation全模块功能验证UVM + SystemVerilog功能覆盖率 + 代码覆盖率
FPGA 原型系统集成验证、软件联调Xilinx/Intel FPGA 平台端到端功能、RTOS 联调
Formal 验证关键属性验证 (安全/一致性)JasperGold / VC FormalPMP 隔离、锁步一致性、CDC
Emulation大规模软件回归Palladium / Veloce (可选)OS Boot、Benchmark 运行

1.2 验证原则

  • 需求驱动：每个 PRD 需求 (≥ BL-1.1, 99 条) 至少对应一个验证用例
  • 分层收敛：单元验证 → 子系统验证 → 系统集成验证，逐层收敛
  • 回归保护：所有通过用例进入回归套件，每日自动化运行
  • 覆盖率量化：代码覆盖率 (Line/Toggle/Branch/Condition) + 功能覆盖率 (Cross Coverage)

5. 回归测试框架

5.1 回归层次

层次频率范围超时
Smoke Test每次提交 (CI)核心用例 Top-50≤30 min
Quick Regression每日 (Nightly)全模块覆盖用例 (~500 条)≤8 hr
Full Regression每周 / 里程碑全部用例 + 覆盖率收集≤72 hr
Performance Regression每月Benchmark 套件运行≤24 hr

5.2 回归规则

  • Smoke Test 失败 → 阻塞提交，需立即修复
  • Quick Regression 新增 Failure → 自动创建 Bug Ticket
  • Full Regression 覆盖率未达标 → 生成 Waiver Report 供评审
  • Performance Regression 性能回退 >2% → 触发性能分析
