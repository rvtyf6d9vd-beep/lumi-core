1. 概述与范围

1.1 模块职责

DFT & 安全模块覆盖 Lumi-Core 的可测试性设计、信息安全和物理约束：

  • Scan Chain：全扫描链插入，≥98% stuck-at fault 覆盖率
  • MBIST：TCM + L1 Cache 内存内建自测试
  • JTAG 边界扫描：IEEE 1149.1 合规，与调试 JTAG 复用物理接口
  • 信息安全：安全启动 ROM、调试安全、PMP 隔离、总线完整性
  • 物理约束：SDC 时序约束、PVT 角落覆盖、IP 质量指标

2. 功能描述

2.1 Scan Chain 扫描链 LUMI-DFT-001 审查约束: 扫描链覆盖率

2.1.1 扫描链设计

参数规格
覆盖率目标≥98% stuck-at fault + ≥95% transition fault
扫描链数量4~8 条 (可配，减少测试时间)
最大链长度≤5000 FF/chain (平衡测试时间)
扫描模式Functional / Scan / MBIST (通过 scan_mode 信号切换)
扫描时钟scan_clk (可选独立时钟，降低测试功耗)
压缩可选 DFT 压缩 (DFTMAX / iJTAG)，减少 I/O 引脚需求

2.1.2 DFT 插入点

// DFT 插入点定义 (RTL 设计预留)
//
// 1. 扫描链使能:
//    - 顶层信号: scan_enable (SE), scan_mode
//    - scan_mode = 1 时: 所有 FF 切换为 scan FF (MUX-based)
//    - scan_enable 控制 shift/capture 模式
//
// 2. 扫描链分组 (默认 4 条):
//    Chain 0: 流水线寄存器 (F1/F2/D/I/E1/E2/M/W)
//    Chain 1: CSR + 中断控制器 (CLIC/PLIC 寄存器)
//    Chain 2: FPU + Vector 单元
//    Chain 3: 内存控制器 (TCM/Cache 控制逻辑, 不含 RAM)
//
// 3. RAM 处理:
//    - TCM/Cache SRAM 不可扫描 (非 FF)
//    - 通过 MBIST 覆盖 (§2.2)
//    - SRAM 读写端口通过扫描链验证
//
// 4. 多时钟域处理:
//    - 跨域 FF 使用 scan-safe 同步器
//    - tck 域 FF 不参与扫描 (独立 JTAG 测试)
审查约束 — 扫描链覆盖率:
  • 目标: ≥98% stuck-at fault (LUMI-DFT-001)
  • 不可扫描: SRAM (TCM/Cache) 通过 MBIST 覆盖; 模拟 IP (PLL) 通过专用测试
  • 覆盖率计算: (可扫描 FF 数 / 总 FF 数) × ATPG 覆盖率
  • 预期: Lumi-Core 可扫描 FF 比例 ≥95% (SRAM 控制逻辑可扫描), ATPG 覆盖率 ≥99%
  • 验证: ATPG 工具生成测试向量后报告实际覆盖率; Cover Group cg_scan_coverage

2.2.2 MBIST 控制器

// MBIST 控制器接口
//
// 信号:
//   mbist_enable    — MBIST 模式使能 (scan_mode 下激活)
//   mbist_start     — 启动测试序列
//   mbist_done      — 测试完成
//   mbist_fail      — 测试失败 (至少一个故障检测到)
//   mbist_addr[31:0] — 故障地址 (诊断用)
//   mbist_data[31:0] — 期望值 / 实际值 (诊断用)
//
// 与 ECC 联调:
//   MBIST 期间禁用 ECC 检查 (mbist_ecc_bypass = 1)
//   避免 ECC 纠正掩盖 SRAM 故障
//
// March C- 算法序列:
//   M0: ↑(w0)                           — 全写 0
//   M1: ↑(r0, w1)                       — 升序读 0 写 1
//   M2: ↑(r1, w0)                       — 升序读 1 写 0
//   M3: ↓(r0, w1)                       — 降序读 0 写 1
//   M4: ↓(r1, w0)                       — 降序读 1 写 0
//   M5: ↑(r0)                           — 升序读 0
//   复杂度: 10N (N = memory depth)

2.3 JTAG 边界扫描 LUMI-DFT-003

参数规格
标准IEEE 1149.1 (JTAG 边界扫描)
物理接口与调试 JTAG 复用 (TCK/TMS/TDI/TDO/TRST)
TAP 指令EXTEST / INTEST / SAMPLE / PRELOAD / BYPASS / IDCODE
BSDL 文件交付完整 BSDL 描述
Boundary Cell 数等于芯片 I/O 引脚数 (含 AXI/JTAG/时钟/复位)
IDCODE32 bit (厂商 ID + 器件型号 + 版本号)

JTAG TAP 与 Debug TAP 复用:

  • 边界扫描 TAP 和 Debug TAP 共享物理 JTAG 引脚
  • 通过 JTAG 指令区分: BYPASS/EXTEST/INTEST → 边界扫描; 自定义指令 → Debug
  • 可选 cJTAG (IEEE 1149.7) 支持 2-pin 模式 LUMI-FEAT-004

2.4 安全启动 ROM LUMI-SEC-001

Boot ROM 固件签名验证（引用 power-reset.html §2.5）：

  • 算法: RSA-2048 或 ECDSA-P256 签名验证
  • 流程: Boot ROM 加载固件 → 读取签名 → 验证公钥 (eFuse/OTP) → 通过则跳转
  • 密码 IP: 外部硬件加速器 (非 Lumi-Core 内置), 通过 AXI 接口调用
  • 安全链: ROM 代码不可修改 → 签名验证不可绕过 → 固件完整性保证

2.5 安全调试策略 LUMI-SEC-003

三级调试安全等级（引用 debug-trace.html §2.6）：

安全等级描述能力
Level 0 (完全锁定)生产环境禁止一切调试操作 (JTAG 不响应, Abstract Command 禁止)
Level 1 (仅断点)受限调试允许硬件断点 + 单步; 禁止寄存器/内存读写
Level 2 (完全开放)开发环境全部调试功能开放

2.6 PMP 安全隔离 LUMI-SEC-004

基于 PMP + Smepmp 的安全隔离（引用 privilege-pmp.html §2.3）：

  • Smepmp 强制生效: M-mode PMP 配置不可被低特权级篡改
  • PMP 锁定 (L=1): 复位前不可修改
  • IP 隔离边界: 基于 PMP 的内存访问控制

2.7 总线完整性保护 LUMI-SEC-005

总线 Parity 保护（引用 pma-axi.html）：

  • AXI 主总线: 数据 Parity 校验 (可选 ECC)
  • FFP 外设总线: 数据 Parity 校验
  • 异常处理: bus_error_exception → 可配策略 (中断/NMI/复位)

2.8 加密加速 LUMI-SEC-002 (Optional)

加密加速通过 Vector Crypto (Zvkg/Zvkn) 或 Packed SIMD 实现（引用 fpu-vector.html §2.4）。

4. 数据通路

4.1 DFT 插入点拓扑

                    ┌─────────────────────────────────────────────┐
                    │              Lumi-Core Top                  │
                    │                                             │
  scan_in[0] ──►   │  ┌─── Chain 0: Pipeline FF (F1~W) ───┐      │
  scan_in[1] ──►   │  ├─── Chain 1: CSR + IRQ (CLIC/PLIC) ┤      │
  scan_in[2] ──►   │  ├─── Chain 2: FPU + Vector ─────────┤──► scan_out[3:0]
  scan_in[3] ──►   │  └─── Chain 3: Mem Ctrl (TCM/Cache) ─┘      │
                    │                                             │
                    │  SRAM (TCM/Cache) ──► MBIST Controller      │
                    │  (不参与扫描, 由 MBIST 覆盖)                 │
                    │                                             │
  tck/tms/tdi ──►  │  JTAG TAP ──► Boundary Scan Cells           │
  tdo ◄────────    │  (与 Debug TAP 复用物理引脚)                 │
                    └─────────────────────────────────────────────┘

扫描链在 scan_mode=1 时激活，所有功能 FF 通过 MUX 切换为 Scan FF。SRAM 不可扫描，通过 MBIST 独立覆盖。JTAG 边界扫描通过 TAP 控制器驱动芯片 I/O 的 Boundary Cell。

6.2 关键验证场景

  • 扫描链功能: 验证 Scan Mode 下 FF 移位/捕获正确; Functional 模式下无功能影响
  • MBIST 故障注入: 在 SRAM 模型中注入 stuck-at/transition 故障，验证 MBIST 检测
  • JTAG 边界扫描: EXTEST 驱动外部引脚 + SAMPLE 采样 + BYPASS 直通
  • 安全调试: Level 0 下禁止所有调试操作; Level 1 仅断点; Level 2 全开放
  • 模式切换: Functional → Scan → Functional 无残留状态
