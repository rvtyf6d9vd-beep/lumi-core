E1/E2 Stage — Integer ALU + Pipelined MUL + Iterative DIV

1. 概述与范围

1.1 模块职责

Execute 模块覆盖 Lumi-Core 8 级流水线的 E1 (Execute 1) 和 E2 (Execute 2) 两级，负责所有整数运算、分支判断和地址计算：

  • E1 级：ALU 运算 (add/sub/and/or/xor/slt/shift)、分支条件判断、Load/Store 地址计算、JALR 目标地址计算
  • E2 级：MUL 流水线执行 (opLat=1)、DIV 迭代执行第一级 (opLat=9)、多周期指令流水线化
GEM5 映射: Spec 的 E1+E2 对应 GEM5 MinorCPU 的 Execute 级。GEM5 通过 opLat 参数建模不同 FU 的执行延迟，ALU/MUL opLat=1 (E1 即完成)，DIV opLat=9 (跨 E1~W 多 cycle)。

2.1.3 分支判断

所有分支指令在 E1 级完成条件判断，误预测时触发 F2/D/I 级 flush：

分支funct3条件误预测惩罚
BEQ000rs1 == rs23 cycle flush
BNE001rs1 != rs23 cycle flush
BLT100rs1 < rs2 (signed)3 cycle flush
BGE101rs1 ≥ rs2 (signed)3 cycle flush
BLTU110rs1 < rs2 (unsigned)3 cycle flush
BGEU111rs1 ≥ rs2 (unsigned)3 cycle flush
分支 FU 复用: 分支指令使用 ALU FU 执行比较操作，不占用独立 FU (Phase 11 优化后取消 Branch FU)。分支目标地址在 F1 级由预测器计算，E1 级验证目标正确性。

2.1.4 JALR 地址计算 D-010

D-010 JALR 解析延迟: JALR 目标地址 = (rs1 + imm) & ~1，需要 D 级读取 rs1 寄存器值 + E1 级计算最终地址。若分支预测器未能准确预测 JALR 目标 (间接跳转)，E1 级检测到误预测后触发 flush。

延迟分析:
• D 级: 读取 rs1 寄存器值 (1 cycle)
• E1 级: 计算 (rs1 + imm) & ~1，与预测目标比较 (1 cycle)
• Branch feedback: 将正确 PC 反馈到 F1 级 (1 cycle, executeBranchDelay=1)
• 总计: 3 cycle 误预测惩罚 (与条件分支相同)
// JALR 目标计算伪代码
function jalr_execute(rs1_val, imm):
    target = (rs1_val + sign_extend(imm)) & 0xFFFFFFFE  // 对齐到 2 字节
    predicted_target = btb_lookup(pc)

    if target != predicted_target:
        // 误预测: flush F2/D/I, 重定向 PC
        flush_pipeline(F2 | D | I)
        redirect_pc(target)
        return MISPREDICT
    else:
        return CORRECT

2.2 E2 级：MUL/DIV 执行

2.2.1 MUL 流水线化乘法 D-014

D-014 MUL 吞吐模型: MUL 采用流水线化乘法器 (Phase 7 优化后)，opLat=1，每个 cycle 可接受 1 条新乘法指令，吞吐 1 条/cycle。独立 MUL FU (LumiIntMulFU)，不与 DIV 共享。
操作funct3funct7描述延迟吞吐
MUL0000000001rs1*rs2 低 32 位1 cycle1/cycle
MULH0010000001有符号*有符号 高 32 位1 cycle1/cycle
MULHSU0100000001有符号*无符号 高 32 位1 cycle1/cycle
MULHU0110000001无符号*无符号 高 32 位1 cycle1/cycle
// MUL 流水线化执行模型
// Phase 7 优化: 从 3-cycle 迭代改为 1-cycle 流水线化 Wallace Tree
pipeline_stage MUL_execute(op, rs1, rs2):
    case op:
        MUL:    result = (rs1 * rs2)[31:0]
        MULH:   result = (signed(rs1) * signed(rs2))[63:32]
        MULHSU: result = (signed(rs1) * unsigned(rs2))[63:32]
        MULHU:  result = (unsigned(rs1) * unsigned(rs2))[63:32]
    // opLat=1: 结果在 E1 结束时即可用, 通过旁路网络转发
    // 下一个 cycle 可接受新的 MUL 指令 (全流水线化)
    return result

2.2.2 DIV 迭代式除法 D-014

D-014 DIV 延迟模型: DIV 采用迭代式除法器 (SRT Radix-4)，opLat=9。执行期间 DIV FU 被独占，但其他 FU (ALU/MUL/LSU/FPU) 可并行执行。skip-stalled 机制允许后续不依赖 DIV 结果的指令越过等待。
操作funct3funct7描述延迟吞吐
DIV1000000001有符号除法商9 cycle1/9 cycle
DIVU1010000001无符号除法商9 cycle1/9 cycle
REM1100000001有符号除法余数9 cycle1/9 cycle
REMU1110000001无符号除法余数9 cycle1/9 cycle
// DIV 执行模型 (SRT Radix-4 迭代)
function div_execute(op, rs1, rs2):
    if rs2 == 0:
        // 除以零: 按 RISC-V 规范返回 -1 (商) 或 rs1 (余数)
        if op in (DIV, DIVU): return 0xFFFFFFFF
        else: return rs1  // REM/REMU

    // 特殊情况: 有符号最小值 / -1
    if op == DIV and rs1 == 0x80000000 and rs2 == 0xFFFFFFFF:
        return 0x80000000  // 溢出, 返回被除数

    quotient, remainder = srt_radix4_divide(rs1, rs2, 32)
    // 32-bit / Radix-4 = 16 迭代, 每次 2 bit
    // 总延迟: 1 (setup) + 7 (迭代) + 1 (finalize) = 9 cycle

    case op:
        DIV:  return sign_adjust(quotient)
        DIVU: return quotient
        REM:  return sign_adjust(remainder)
        REMU: return remainder

2.2.3 DIV 与 skip-stalled 协同

DIV 执行 9 cycle 期间，DIV FU 被独占。后续指令的发射行为：

场景skip-stalled 行为影响
后续指令不依赖 DIV 结果 + 不使用 DIV FU正常发射到 ALU/LSU/FPU0 stall
后续指令需要 DIV FU (第二条 DIV)skip-stalled: 跳过该指令，发射后续非 DIV 指令该指令 stall 直到 DIV FU 释放
后续指令 RAW 依赖 DIV rdstall 等待 DIV 结果最多 8 cycle stall (DIV 剩余 cycle)

2.3 E1 级分支反馈路径

分支指令在 E1 级完成后，结果反馈到 F1 级用于 PC 修正：

参数值描述
executeBranchDelay1 cycleE1 到 F1 的反馈延迟 (GEM5 Phase 5 优化)
Flush 范围F2 + D + I (3 级)误预测时清空预测路径上所有指令
Flush 延迟1 cycle (下一 cycle 从正确 PC 取指)含在 3 cycle 惩罚内
// 分支解析与 flush 伪代码
function branch_resolve(alu_result, pred_taken, pred_target, pc):
    actual_taken = (alu_result == 0) ? NOT_TAKEN : TAKEN  // 比较结果为 0 表示条件满足
    actual_target = pc + sign_extend(imm)

    if actual_taken != pred_taken or (actual_taken and actual_target != pred_target):
        // 误预测
        flush_stages(F2 | D | I)
        if actual_taken:
            redirect_pc(actual_target)
        else:
            redirect_pc(pc + 4)  // 顺序执行
        increment_hpm(BRANCH_MISS)
    else:
        // 预测正确, 无需 flush
        pass

4. 数据通路

4.1 E1/E2 级数据流

I 级发射 ──→ E1 级 ALU 运算 ──→ 结果旁路到 D 级 (E1→D)
     │              │
     │              ├─ ALU 结果 ──→ E2 ──→ M ──→ W (写回)
     │              ├─ Branch 判断 ──→ 反馈到 F1 (branch feedback)
     │              ├─ Mem Addr ──→ LSU (Load/Store 执行)
     │              └─ JALR 地址 ──→ 误预测检测 ──→ flush
     │
     └──(MUL/DIV 指令)──→ E2 级 MUL/DIV 执行
                              │
                              ├─ MUL 结果 (1 cycle) ──→ M ──→ W
                              └─ DIV 结果 (9 cycle) ──→ M ──→ W

4.2 FU 分配逻辑

// FU 分配伪代码 (I 级执行)
function fu_dispatch(decoded_inst):
    case decoded_inst.type:
        ALU_OP:     // add/sub/and/or/xor/slt/shift/B-ext
            assign to free LumiIntFU (4 available)
        MUL_OP:     // mul/mulh/mulhsu/mulhu
            assign to LumiIntMulFU (1 available)
        DIV_OP:     // div/divu/rem/remu
            if div_busy: stall (or skip-stalled)
            else: assign to MinorDefaultIntDivFU
        BRANCH_OP:  // beq/bne/blt/bge/bltu/bgeu
            assign to free LumiIntFU (复用 ALU 做比较)
        MEM_OP:     // load/store
            assign to free MinorDefaultMemFU (2 available)
        FPU_OP:     // fadd/fmul/fdiv/...
            assign to MinorDefaultFloatSimdFU
        MISC_OP:    // ecall/ebreak/fence
            assign to MinorDefaultMiscFU

4.3 旁路网络 (Execute 侧输出)

E1/E2 级的执行结果通过旁路网络转发到 D 级，消除 RAW 冒险：

旁路路径数据源可用时机延迟
E1→DALU/Branch 运算结果E1 级结束 (当前 cycle)0 cycle (下条指令 D 级即可用)
E2→DMUL 最终结果E2 级结束 (E1 + 1 cycle)0 cycle (MUL opLat=1, E1 即可用)
DIV 结果DIV 最终结果DIV 完成 (9 cycle 后)等待 DIV 完成
RISK-011 旁路 mux 关键路径: 5 路旁路 mux (E1/E2/M/W/FPU→D) 的 5-to-1 选择器延迟约 80~100ps，占 E1/I 级 125ps 预算的 64~80%。RTL 实现时需评估是否需要在 mux 中间插入流水寄存器分割关键路径。
