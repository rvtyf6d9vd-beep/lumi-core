4. 状态机定义

本模块无显式多状态 FSM。异常处理为单 cycle 组合逻辑决策， 仅有 NMI 边沿锁存使用 2 个触发器 (nmi_latch, nmi_prev)。

4.1 NMI 锁存状态模型

状态名条件含义进入条件
NMI_CLEARnmi_latch == 0无 NMI pendingreset / NMI trap 进入 / MNRET 执行
NMI_LATCHEDnmi_latch == 1NMI 已锁存，等待处理nmi_signal 上升沿

4.2 NMI 锁存转移图

stateDiagram-v2
    [*] --> NMI_CLEAR : reset
    NMI_CLEAR --> NMI_LATCHED : nmi_signal 上升沿 (nmi_sync && !nmi_prev)
    NMI_LATCHED --> NMI_CLEAR : nmi_enter_out (trap 处理) || mnret_exec_out (返回)
    NMI_LATCHED --> NMI_LATCHED : 保持 (未处理)
    NMI_CLEAR --> NMI_CLEAR : 无 NMI

4.3 异常处理优先级 (隐式状态)

优先级事件类型触发条件输出动作
1 (最高)MNRETis_mnret == 1trap_pc=mnepc_in, mnret_exec_out=1
2MRETis_mret == 1mret_exec=1, trap_pc=0 (CSR 提供 mepc)
3SRETis_sret == 1sret_exec=1, trap_pc=0 (CSR 提供 sepc)
4同步异常has_exception == 1trap_enter=1, cause=exc_cause_sel, tval=exc_tval_sel
5NMInmi_latch == 1nmi_enter_out=1, cause=0x80000000
6 (最低)普通中断irq_request && mie_enabledtrap_enter=1, cause=irq_id

4.4 死锁/活锁分析

  • 死锁防护: 无 FSM 循环，所有 trap 事件单 cycle 完成。NMI 锁存确保不丢失脉冲信号。
  • 活锁防护: 返回指令 (MRET/SRET/MNRET) 具有最高优先级，确保 trap 返回不会被新 trap 阻塞。
  • NMI 互斥: NMI 处理期间 (nmi_latch=1 且已进入 trap)，nmi_enter_out 或 mnret_exec_out 清除锁存，新 NMI 可被重新锁存。

6.3 未定义行为声明

  • exc_decode 中 4'b0110~4'b1111 为未定义编码，decode_exc_valid=0 (default 分支)
  • exc_exec 中 3'b101~3'b111 为未定义编码，exec_exc_valid=0
  • MRET/SRET/MNRET 与异常同时发生时，返回指令优先 (异常被忽略)。HLD 中返回指令不应在有 pending 异常时执行
  • nmi_signal 为异步信号，RTL 中未同步 (直通)；亚稳态传播为未定义行为

8.3 关键算法

// 异常处理完整优先级链 (组合逻辑 always_comb)
// 优先级: MNRET > MRET > SRET > 同步异常 > NMI > 中断

if is_mnret:
    trap_take=1, trap_pc=mnepc_in, mnret_exec_out=1, flush_pipe=1

elif is_mret:
    trap_take=1, trap_pc=0(CSR提供mepc), mret_exec=1, flush_pipe=1

elif is_sret:
    trap_take=1, trap_pc=0(CSR提供sepc), sret_exec=1, flush_pipe=1

elif has_exception:  // 同步异常
    trap_take=1, trap_cause={27'b0, exc_cause_sel}
    trap_tval=exc_tval_sel, trap_epc=exc_pc
    trap_enter=1, flush_pipe=1
    if delegate_to_s: trap_pc=trap_target_s
    else:             trap_pc=trap_target_m

elif nmi_latch:  // NMI
    trap_take=1, trap_cause=32'h80000000
    trap_epc=exc_pc, nmi_enter_out=1, flush_pipe=1
    trap_pc=trap_target_m  // NMI 始终 M-mode

elif irq_request && mie_enabled:  // 普通中断
    trap_take=1, trap_cause={24'b0, irq_id}
    trap_epc=exc_pc, trap_enter=1, flush_pipe=1
    if delegate_to_s: trap_pc=trap_target_s
    else:             trap_pc=trap_target_m
