4.4 优先级仲裁

在 ST_IDLE 状态，请求优先级为：

  1. FENCE.I (fence_i_req) — 最高优先级， invalidate I-Cache 保证指令流一致
  2. FENCE (fence_req) — drain store buffer
  3. I-Cache (ic_valid) — 取指请求
  4. D-Cache (dc_valid) — 数据访问 (最低优先级)

4.5 死锁/活锁分析

  • 死锁防护: AXI 握手协议保证 valid 持续拉高直到 ready 响应。Refill/Evict FSM 无超时机制，依赖 AXI slave 最终响应。系统级需保证 AXI interconnect 无死锁。
  • 活锁防护: IDLE 优先级仲裁为静态优先级 (FENCE.I > FENCE > IC > DC)，不存在活锁风险。D-Cache 不会饿死因为 FENCE/I-Cache 操作为有限状态 (1 cycle 即返回 IDLE)。
  • Refill/Evict 互斥: 共享 AXI AR/R 通道，FSM 保证 I-Cache refill 与 D-Cache refill 不会同时发起 (串行化于 IDLE)。

5.3 PLRU 替换算法

// ─── PLRU 二叉树结构 (4-way, 3 bit/set) ───────────────
//
//         b0
//        /  \
//      b1    b2
//     / \   / \
//   W0  W1 W2  W3
//
// ─── 8-way 扩展 (7 bit/set) ───────────────────────────
//
//             b0
//           /    \
//        b1        b2
//       /  \      /  \
//     b3    b4  b5    b6
//    / \  / \  / \  / \
//  W0 W1 W2 W3 W4 W5 W6 W7
//
// bit=0 → 左子树 (最近访问方向)
// bit=1 → 右子树
// Victim: 从根沿 bit 方向走到叶子

// RTL 实现 (lumi_cache.sv L182-L211):
function plru_victim(plru_bits):
    way = 0
    for level = 0 to log2(WAYS)-1:
        bit_idx = ((1 << level) - 1) + (way >> (log2(WAYS) - 1 - level))
        way = (way << 1) | plru_bits[bit_idx]
    return way

// Victim 选择优先级: invalid way 优先于 PLRU victim
// (lumi_cache.sv L254-L272):
//   1. 遍历所有 way, 找首个 valid=0 的 way
//   2. 若全部 valid=1, 使用 PLRU victim

6.2 错误检测与恢复

错误类型检测方法恢复策略
Data RAM SEC (单 bit 错误)ECC 解码 (SECDED, 预留端口)自动纠正 + 回写 + dc_ecc_ce_irq=1
Data RAM DED (双 bit 错误)ECC 解码 (SECDED, 预留端口)触发 Load/Store Access Fault + dc_ecc_ded_irq=1

6.3 未定义行为声明

  • 多 hit 异常: 若同一 set 中多个 way 同时 hit (不应发生, 表示 tag RAM 损坏), RTL 取最高编号 way (ic_hit_way 在 for 循环中最后赋值覆盖)。实际应通过 ECC 检测。
  • ic_flush 端口: 当前 RTL 声明了 ic_flush 输入端口但未在 FSM 中直接使用 (FENCE.I 通过 fence_i_req 触发)。该端口为预留接口。
  • AXI BRESP: Evict 写回不检查 AXI B 通道响应 (无 evict_bvalid/bresp 端口), 假设写回总是成功。

6.4 Stall / Flush / Reset 响应

全局信号响应行为时序
ic_ready=0 (miss stall)I-Cache miss 时拉低 ready, 流水线 F1 级 stall (PC 冻结)同 cycle 生效 (combinational)
dc_ready=0 (miss stall)D-Cache miss 时拉低 ready, M 级 stall (pipeline back-pressure)同 cycle 生效 (combinational)
fence_i_req=1Invalidate 全部 I-Cache lines (valid=0), 触发 pipeline flush下一 clk↑ 生效 (tag RAM 写入)
reset_n=0异步复位所有寄存器: FSM→IDLE, 计数器清零。Tag/Data RAM 不复位 (cold start)异步

7.2 参数约束关系

  • ICACHE_SIZE 必须 ≥ ICACHE_WAYS × ICACHE_LINE (至少 1 set)
  • DCACHE_SIZE 必须 ≥ DCACHE_WAYS × DCACHE_LINE (至少 1 set)
  • ICACHE_WAYS / DCACHE_WAYS 必须为 2 的幂 (PLRU 二叉树要求)
  • ICACHE_LINE / DCACHE_LINE 必须为 2 的幂 (地址解码要求)
  • ICACHE_LINE / DCACHE_LINE 必须 ≥ 16 (AXI 128bit = 16B/beat, 至少 1 beat)
  • IC_IDX_W + IC_OFF_W + IC_TAG_W = 32 (32bit 地址空间, 由 RTL 自动保证)

8.3 关键算法 — Tag 比较与数据提取

// ─── I-Cache Tag 并行比较 (组合逻辑, L224-L236) ───────
always_comb begin
    ic_any_hit = 0; ic_way_hit = 0; ic_hit_way = 0;
    for (w = 0; w < ICACHE_WAYS; w++):
        if ic_tag_mem[ic_idx][w].valid && ic_tag_mem[ic_idx][w].tag == ic_tag:
            ic_way_hit[w] = 1
            ic_any_hit    = 1
            ic_hit_way    = w        // 最后 hit 覆盖 (多 hit 取最高 way)

// ─── I-Cache 128bit 数据提取 (L275-L283) ─────────────
// aligned_off 对齐到 16B 边界: offset[5:4] << 4
ic_rdata = ic_data_mem[ic_idx][ic_hit_way][(aligned_off*8) +: 128]

// ─── D-Cache 32bit word 提取 (L286-L292) ─────────────
// word_off 对齐到 4B 边界: offset[5:2] << 2
dc_rdata = dc_data_mem[dc_idx][dc_hit_way][(word_off*8) +: 32]

// ─── D-Cache Store Hit 字节写入 (L362-L369) ──────────
// 按 dc_be[3:0] 逐字节写入 data RAM
for (byt = 0; byt < 4; byt++):
    if dc_be_reg[byt]:
        dc_data_mem[dc_idx][dc_hit_way][byte_offset * 8 +: 8] <= dc_wdata_reg[byt*8 +: 8]
dc_tag_mem[dc_idx][dc_hit_way].dirty <= 1
