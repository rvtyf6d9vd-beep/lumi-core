1. 概述与范围

1.1 模块职责

Writeback & CSR/HPM 模块覆盖 Lumi-Core 8 级流水线的 W (Writeback) 级，以及全局 CSR 寄存器文件和硬件性能监控计数器：

  • W 级：运算结果写回整数/浮点/向量寄存器堆、指令提交 (in-order commit)、异常/中断检测与触发
  • CSR 文件：RISC-V 标准 CSR 寄存器 (mstatus/mtvec/mepc/mcause 等) 的读写逻辑
  • HPM：Zicntr 基础计数器 + Zihpm 性能监控计数器
GEM5 映射: W 级对应 GEM5 MinorCPU 的 Writeback 级。CSR 操作通过 MISC FU (opLat=1) 执行，GEM5 建模为系统指令的写回动作。

2. 功能描述

2.1 W 级：指令提交与写回

2.1.1 In-Order Commit

W 级是流水线的最终级，负责按程序顺序提交指令。所有指令必须通过 W 级才能产生可见的架构状态改变：

动作条件描述
寄存器写回rd != x0 且指令有写回目标ALU/MUL/DIV/Load 结果写入整数堆; FPU 结果写入浮点堆
Store 提交Store 指令到达 W 级Store 数据写入 DCache/TCM (已在 M 级准备，W 级确认)
CSR 更新SYSTEM 指令CSRRW/CSRRS/CSRRC 写回 CSR 文件
异常触发异常条件成立中止当前指令提交，进入 trap 处理
中断检测mip & mie & mstatus.MIE 非零当前指令提交后进入中断处理

2.1.2 写回端口仲裁

整数寄存器堆 2W 端口，最多同时 2 条指令写回：

// 写回端口仲裁
function writeback_arbitrate(slot_results[0..2]):
    write_ports = 2  // 2W ports for integer register file
    for i in range(min(len(slot_results), write_ports)):
        if slot_results[i].valid and slot_results[i].rd != 0:
            regfile_write(slot_results[i].rd, slot_results[i].data)
            write_ports -= 1

    // 超过 2W 端口的情况 (三发射理论上最多 3 条同时到达 W 级)
    // 实际上: 由于 E1/E2 延迟差异，ALU (1c) 和 DIV (9c) 不会同时到达
    // 最坏情况: 2 条 ALU + 1 条 Load 同时 W 级 → 需要仲裁
    if write_ports < remaining_writes:
        stall_pipeline(1)  // 延迟 1 cycle
写端口充足性分析: 三发射模式下，最多 3 条指令可能在同一 cycle 到达 W 级。但由于不同 FU 的延迟差异 (ALU=1c, MUL=1c, DIV=9c, Load=1c+, FPU=6c)，实际同时到达 W 级最多 2 条 (如 1 条 ALU + 1 条 Load)。极端场景 (3 条 ALU 指令同时发射、同时完成) 需要 3W 但仅 2W 可用，此时 stall 1 cycle。2W 端口满足绝大多数场景。

2.1.3 异常与中断检测

W 级是异常和中断的 精确检测点，保证精确异常 (Precise Exception)：

检测项时机处理
指令访问异常F2 级已标记，W 级确认提交触发 trap
非法指令D 级已标记，W 级确认提交触发 trap (mcause=2)
Load/Store 异常M 级已标记，W 级确认提交触发 trap (mcause=5/7)
ECALLW 级检测到 SYSTEM 指令触发 trap (mcause=11)
EBREAKW 级检测到 SYSTEM 指令进入 Debug 模式或触发 trap (mcause=3)
中断W 级指令提交后检查 mip&mie当前指令提交后进入 trap
// W 级异常/中断检测伪代码
function writeback_commit(inst):
    if inst.exception_pending:
        // 异常优先于中断
        enter_trap(inst.exception_cause, inst.pc)
        flush_all_stages()
        return

    // 正常提交指令
    commit_instruction(inst)
    if inst.has_writeback and inst.rd != 0:
        regfile_write(inst.rd, inst.result)
    increment_hpm(INST_RETIRED)

    // 指令提交后检查中断
    if (mip & mie & mstatus.MIE) != 0:
        highest_irq = find_highest_priority_irq(mip & mie)
        if highest_irq.priority > mstatus.MIL:  // CLIC 优先级比较
            enter_trap(IRQ_BASE + highest_irq.id, pc_next)
            flush_all_stages()

2.2.3 Trap CSR 自动更新

异常/中断触发时，硬件自动更新以下 CSR：

CSR更新内容时机
mepc触发异常/中断的指令 PCtrap 入口
mcause异常原因码 (最高位: 0=异常, 1=中断)trap 入口
mtval异常附加信息 (如非法指令字、访问地址)trap 入口
mstatus.MPIE← mstatus.MIE (保存中断使能)trap 入口
mstatus.MIE← 0 (禁止中断)trap 入口
mstatus.MPP← 当前特权级 (M-mode)trap 入口
mintstatusCLIC 中断状态 (若 CLIC 中断)CLIC trap 入口
// Trap 入口 CSR 更新伪代码
function enter_trap(cause, pc):
    mepc = pc
    mcause = cause
    mtval = trap_value(cause)  // 指令字或访问地址

    mstatus.MPIE = mstatus.MIE
    mstatus.MIE = 0
    mstatus.MPP = current_privilege

    // 跳转到 mtvec
    if mtvec.MODE == VECTORED and is_interrupt(cause):
        new_pc = mtvec.BASE + 4 * exception_code(cause)
    else:
        new_pc = mtvec.BASE  // 直接模式

    redirect_pc(new_pc)

2.3.2 Zihpm 扩展计数器

29 个可编程性能计数器 (mhpmcounter3~31)，通过 mhpmevent3~31 选择事件源：

事件 ID事件名称来源计数方式
0x00CPU_CYCLES全局时钟= mcycle
0x01INST_RETIREDW 级提交= minstret
0x02BRANCH_MISSE1 级误预测每次 flush +1
0x03I_CACHE_MISSF2 级 ICache miss每次 ICache miss +1
0x04D_CACHE_MISSM 级 DCache miss每次 DCache miss +1
0x05EXCEPTION_TAKENW 级 trap每次异常 +1

// HPM 计数器更新伪代码 (每 cycle)
function hpm_update():
    mcycle += 1  // 始终计数

    if instruction_committed:
        minstret += commit_count  // 可能 1~3 条

    // 可编程计数器
    for i in range(3, 32):
        event_id = mhpmevent[i] & 0x3FF  // 10-bit 事件 ID
        if event_match(event_id):
            mhpmcounter[i] += 1

2.3.3 RV32 高/低 32 位拆分访问

RV32 架构下 64-bit 计数器需要两次读取 (低 32 位 + 高 32 位)，为避免读写之间的溢出竞争：

// 64-bit 计数器安全读取 (RV32)
function read_counter_64(lo_addr, hi_addr):
    hi1 = csr_read(hi_addr)
    lo  = csr_read(lo_addr)
    hi2 = csr_read(hi_addr)
    if hi1 != hi2:
        // 高位溢出, 重读
        lo = csr_read(lo_addr)
        hi = hi2
    return (hi << 32) | lo

4. 数据通路

4.1 W 级数据流

M 级输出 ──→ W 级提交 ──→ 寄存器写回 (2W 端口)
                  │
                  ├─ 异常检测 ──→ Trap 处理 (CSR 更新 + PC 跳转)
                  ├─ 中断检测 ──→ 当前指令提交后 Trap
                  ├─ HPM 更新 ──→ mcycle/minstret/event 计数器
                  └─ Store 确认 ──→ DCache/TCM 写确认

4.2 CSR 读写路径

// CSR 读路径
csr_rdata = csr_file[csr_addr]

// CSR 写路径 (原子操作)
case csr_op:
    WRITE (CSRRW/CSRRWI): csr_file[addr] = wdata
    SET   (CSRRS/CSRRSI): csr_file[addr] |= wdata
    CLEAR (CSRRC/CSRRCI): csr_file[addr] &= ~wdata

// 特殊 CSR 行为
if csr_addr == 0x300 (mstatus):
    // mstatus 部分位为 WARL/WLRL, 写入时忽略非法值
    csr_file[0x300] = apply_warl_mask(wdata)
