1. 概述与范围

1.1 模块职责

FPU & Vector 模块实现 Lumi-Core 的浮点运算和向量处理能力：

  • FPU (F/D 扩展)：单/双精度浮点运算，IEEE 754-2008 合规
  • Vector 单元 (V 1.0)：RISC-V 向量扩展，VLEN=128/256 bit，含 zvfh/zvfbf
  • FPU 流水线：6 级流水线化 (opLat=6)，支持 FMA/除法/平方根多周期
  • Vector 执行：分步执行模型 (SEW/LMUL 自适应)，多周期占用 Vector FU
  • 可选扩展：Vector Crypto (zvkg/zvkn) + Packed SIMD (v0.9.4)

2.1.2 FPU 流水线化

// FPU 6 级流水线
//
// Stage 1: 操作数准备 (unpack, NaN-Box check, special case detect)
// Stage 2: 指数对齐 (FADD) / Booth 编码部分积 (FMUL)
// Stage 3: 尾数加法 (FADD) / 部分积累加 (FMUL)
// Stage 4: 归一化 (FADD/FMUL) / SRT 迭代步 (FDIV/FSQRT)
// Stage 5: 舍入 (round + overflow/underflow check)
// Stage 6: Pack + 异常标志生成 + 写回

// FMA (Fused Multiply-Add) 流水线
// f[rd] = f[rs1] * f[rs2] + f[rs3]
// 6 级流水线: unpack → multiply → align+add → normalize → round → pack
// 关键: 中间结果不 round, 仅在最终 stage 做一次舍入 (精确 FMA)

// FDIV / FSQRT (非流水线化, 迭代式)
// SRT radix-4 除法: 12~28 cycle (依赖精度和操作数值)
// FDIV: ~14 cycle (FP32), ~28 cycle (FP64)
// FSQRT: ~18 cycle (FP32), ~28 cycle (FP64)
// 迭代期间 FPU FU busy, 不接受新 FPU 指令 (stall 后序 FPU 指令)
// 非 FPU 指令 (整数/分支) 可在其他 FU 上继续执行

2.2.2 Vector 执行模型 D-006a: Vector-FU 交互

// Vector 指令分步执行模型
//
// 数据通路宽度: 128 bit (VLEN=128 时 1 步完成, VLEN=256 时 2 步)
//
// 每条向量指令的执行:
// 1. Issue 级发射向量指令到 Vector FU
// 2. Vector FU 计算所需步数: steps = ceil(VL * SEW / datapath_width) / LMUL
// 3. 逐步执行, 每步处理 datapath_width 位数据
// 4. 步间可 chaining: 前一步结果可在下一步作为源操作数使用
// 5. 最后一步完成后, Vector FU 释放 busy 信号

// Vector FU busy 信号握手 (DEC-007-2)
function vector_issue_handshake(inst):
    // Issue 级向 Vector FU 发送指令
    if vector_fu_busy:
        return STALL  // 同类型 FU 阻塞, 后续向量指令等待
    
    // 计算执行步数
    steps = compute_steps(inst.vl, inst.sew, inst.lmul)
    vector_fu_busy = 1
    vector_fu_steps_remaining = steps
    
    return ACCEPT

// 每步完成后
function vector_step_complete():
    vector_fu_steps_remaining -= 1
    if vector_fu_steps_remaining == 0:
        vector_fu_busy = 0  // 释放 FU, 可接受新指令
        // 写回结果 (如果是向量运算)
        vector_writeback()
审查约束响应 — D-006a (Vector-FU 交互):
  • 执行延迟: 与 VLEN/SEW/LMUL 相关，steps = ceil(VL * SEW / 128) / LMUL
  • Stall 策略: Vector FU busy 时后续向量指令 stall；整数/标量 FPU 指令可继续在其他 FU 执行
  • Busy 握手: Issue 级检测 vector_fu_busy 信号，busy=1 时不发射新向量指令
  • Chaining: 相邻向量指令若有 RAW 依赖但目标寄存器不同，可通过 chaining 重叠执行

2.2.3 Vector Load/Store D-006b: Vector-LSQ 交互

// Vector Load/Store 地址生成
//
// 三种向量内存操作:
// 1. Unit-stride (连续): 地址 = base + element_index * SEW/8
// 2. Stride (步长): 地址 = base + element_index * stride
// 3. Indexed (scatter/gather): 地址 = base + vs2[element_index]

// Vector LSQ 设计
//
// 向量 Load/Store 使用独立的 Vector LSQ (V-LSQ), 与标量 LSQ 分离:
// - V-LSQ 深度: 8 entry (可配)
// - 每个 entry 跟踪一条向量内存指令的全部 element 访问
// - 向量地址生成单元 (V-AGU): 逐步计算每个 element 的物理地址
//
// V-AGU 地址生成伪代码:
function vector_address_gen(inst):
    base = rs1_value  // 基址 (标量寄存器)
    
    for step in 0..steps-1:
        for elem in step_elements:
            case inst.type:
                UNIT_STRIDE: addr[elem] = base + elem * (SEW/8)
                STRIDE:       addr[elem] = base + elem * stride_value
                INDEXED:      addr[elem] = base + vs2_data[elem]  // vs2 从向量寄存器读
            
            // 发送到 Memory 子系统 (DCache/TCM)
            memory_request(addr[elem], inst.op, SEW/8)
审查约束响应 — D-006b (Vector-LSQ 交互):
  • Scatter/Gather: Indexed Load/Store 通过 V-AGU 逐个计算地址，每个 element 独立发起内存访问
  • LSQ 分离: 向量使用独立 V-LSQ (8 entry)，标量使用 Scalar LSQ；两者通过仲裁共享 DCache 端口
  • V-AGU: 专用地址生成单元，每 cycle 生成 1 个 element 地址 (VLEN/SEW 个 cycle 完成全部)
  • 冲突处理: 向量与标量同时访问 DCache 时，标量优先 (保证标量流水线不被长时间阻塞)

2.2.4 Vector 寄存器堆冲突 D-006c: Vector RegFile 冲突

// 向量寄存器堆 Bank 结构
//
// VLEN=128, 4 banks:
//   Bank 0: v0, v4, v8,  v12, v16, v20, v24, v28
//   Bank 1: v1, v5, v9,  v13, v17, v21, v25, v29
//   Bank 2: v2, v6, v10, v14, v18, v22, v26, v30
//   Bank 3: v3, v7, v11, v15, v19, v23, v27, v31
//
// VLEN=256, 4 banks:
//   每 Bank 存储 8 个向量寄存器的 256bit 数据
//
// LMUL>1 时的 Bank 映射:
//   LMUL=2: v0-v1 组成一组, 跨 Bank 0+1 并行读写
//   LMUL=4: v0-v3 组成一组, 跨 Bank 0+1+2+3 并行读写
//   LMUL=8: v0-v7 组成一组, 全部 4 Bank 并行 (最大并行度)

// 跨 Bank 冲突策略
function vector_regfile_access(vs1, vs2, vd, lmul):
    banks_src1 = get_banks(vs1, lmul)
    banks_src2 = get_banks(vs2, lmul)
    banks_dst  = get_banks(vd, lmul)
    
    // 检测 Bank 冲突
    conflict = false
    if banks_src1 ∩ banks_src2 ≠ ∅:
        conflict = true  // 两个源操作数访问同一 Bank
    if banks_src1 ∩ banks_dst ≠ ∅:
        conflict = true  // 源和目的访问同一 Bank (WAR 场景)
    
    if conflict:
        // 分 2 个 cycle 完成: 先读 src1, 再读 src2/写 dst
        return SPLIT_ACCESS  // 性能损失 1 cycle
    else:
        // 同一 cycle 内并行访问不同 Bank
        return PARALLEL_ACCESS
审查约束响应 — D-006c (Vector RegFile 冲突):
  • Bank 结构: 4 banks (默认)，每 Bank 1R/1W 端口，支持跨 Bank 并行
  • Bank 数与 VLEN: VLEN=128 时 4 banks 足够 (32bit/bank/step)；VLEN=256 时可选 8 banks
  • 冲突策略: 同 Bank 访问分 2 cycle (SPLIT_ACCESS)；不同 Bank 并行 (0 penalty)
  • LMUL 影响: LMUL 越大，跨 Bank 并行度越高，但可用寄存器组数减少
  • 仲裁: 读端口冲突时 src1 优先；写端口冲突时延迟 1 cycle 写入

2.3 DEC-007-2: FPU/Vector 多周期同步接口

// FPU/Vector ↔ Core 同步接口定义
//
// === FPU 接口 ===
// Out: fpu_busy          — FPU 正在执行多周期操作 (FDIV/FSQRT)
// In:  fpu_issue_valid   — Issue 级发射新 FPU 指令
// Out: fpu_issue_ready   — FPU 可接受新指令 (busy=0 或流水线有空位)
// Out: fpu_result_valid  — FPU 结果就绪 (W 级写回)
// Out: fpu_result_data[63:0] — FPU 运算结果
// Out: fpu_fflags[4:0]  — 异常标志 (NX/UF/OF/DZ/IV)
//
// === Vector 接口 ===
// Out: vector_fu_busy    — Vector FU 正在执行多步操作
// In:  vector_issue_valid — Issue 级发射新 Vector 指令
// Out: vector_issue_ready — Vector FU 可接受新指令
// Out: vector_result_valid — Vector 结果就绪
// Out: vector_vstart     — 异常中断时的 vstart 值 (精确恢复)
//
// === 同步规则 ===
// 1. FPU busy 回传 Issue 级: 同类 FU stall, 其他 FU 不受影响
// 2. Vector busy 回传 Issue 级: 仅向量指令 stall, 标量指令继续
// 3. FPU/Vector 异常 (fflags) 在 W 级合并到 fflags CSR
// 4. FPU 与 Vector 共享 fflags CSR: 两者异常标志取 OR
审查约束响应 — DEC-007-2 (FPU/Vector 同步接口):
  • 接口定义: fpu_busy + vector_fu_busy 双向握手，Issue 级 stall 逻辑明确
  • 与 Issue 级连接: fpu_issue_ready = !fpu_busy || fpu_pipeline_has_slot; vector_issue_ready = !vector_fu_busy
  • fflags 合并: FPU 和 Vector 各自产生 fflags，W 级取 OR 合并写入 CSR
  • 精确异常: Vector 异常 (如 element 访问错误) 记录 vstart，支持精确恢复

2.4.2 Packed SIMD LUMI-ISA-010 (Optional)

Packed SIMD 扩展 (v0.9.4) 在标量整数寄存器上执行 SIMD 运算，利用 FPU 数据通路：

  • 操作: 2×16bit / 4×8bit 整数 SIMD 加减乘
  • 实现: 复用 FPU 数据通路 (FMUL 阵列)，在 FPU 内新增 SIMD 子单元
  • 寄存器: 使用浮点寄存器堆 (f0~f31) 存储 SIMD 数据

4.3 FPU↔整数寄存器堆数据通路

FMV/FCVT 指令需要 FPU 与整数寄存器堆之间的数据搬运：

  • fmv.x.w：FP reg → Int reg (1 cycle, 直接连线)
  • fmv.w.x：Int reg → FP reg (1 cycle, NaN-Box 扩展到 64bit)
  • fcvt.w.s/d：FP → Int 转换 (3 cycle, FPU 内完成)
  • fcvt.s/d.w/wu：Int → FP 转换 (3 cycle, FPU 内完成)
  • 旁路路径: FPU→D 旁路 (W 级 FPU 结果直接转发到 D 级)，已在 platform-spec §4.6.1 定义 (DEC-004)

6.2 关键验证场景

  • FPU 精度: IEEE 754 合规测试套件 (全舍入模式 × 全特殊值)
  • Vector 正确性: RVV 1.0 合规测试 (全 SEW × LMUL × VL 组合)
  • 多周期交互: FDIV 执行期间标量指令继续 + 向量指令等待
  • V-LSQ 仲裁: 向量 Load 与标量 Store 同时访问 DCache
  • Bank 冲突: 构造 vs1/vs2/vd 在同一 Bank 的最坏场景
  • 精确异常: 向量执行中发生异常 → vstart 正确记录 → 恢复执行
