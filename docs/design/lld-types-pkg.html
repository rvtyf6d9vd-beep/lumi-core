3.2 跨时钟域接口

Interface 定义本身不处理 CDC, 由各使用模块实现同步逻辑。关键跨时钟域边界:

  • jtag_if: JTAG_TCK ↔ CLK_CORE (需 2-FF 同步)
  • axi4_if: CLK_CORE ↔ CLK_BUS (需异步 FIFO, 由 u_axi wrapper 实现)

5.3 c_ext_expand 函数数据流

// RV32C 压缩指令展开器 (ERR-042: 从 decode_issue 提取到 pkg)
// 输入: 16-bit 压缩指令 ci[15:0]
// 输出: 32-bit 等效指令
//
// 支持象限:
//   Quadrant 0 (ci[1:0]=00): C.ADDI4SPN, C.LW, C.SW
//   Quadrant 1 (ci[1:0]=01): C.ADDI, C.JAL, C.LI, C.ADDI16SP/C.LUI,
//                             C.SRLI/C.SRAI/C.ANDI/C.SUB/C.XOR/C.OR/C.AND,
//                             C.J, C.BEQZ, C.BNEZ
//   Quadrant 2 (ci[1:0]=10): C.SLLI, C.LWSP, C.JR/C.MV/C.EBREAK/C.JALR/C.ADD,
//                             C.SWSP
//
// 关键: rs1p/rs2p = {2'b01, ci[9:7] / ci[4:2]} (寄存器 x8-x15)

6. 边界条件与异常处理

6.1 边界条件清单

编号条件处理策略验证覆盖
BC-001c_ext_expand 非法压缩指令 (default 分支)返回 32'h0000_0000 (全零, 将触发 illegal instruction 异常)非法指令测试
BC-002c_ext_expand C.ADDI16SP vs C.LUI 区分rd_rs1==5'd2 时为 C.ADDI16SP (操作 sp), 否则为 C.LUIsp 操作 vs LUI 测试
BC-003c_ext_expand C.JR vs C.MV vs C.EBREAK vs C.JALR vs C.ADDinst[12]=0: inst[6:2]==0 → C.JR; else → C.MV. inst[12]=1: inst[6:2]==0 && rd==0 → C.EBREAK; inst[6:2]==0 && rd≠0 → C.JALR; else → C.ADD全部 Q2 funct4=100 子类型
BC-004exc_type_e 编码不连续 (4'd10 和 4'd14 缺失)RISC-V 规范: 10 和 14 为保留值, 不应出现在正常异常流中异常码全覆盖测试
BC-005fu_type_e FU_FP(4) 和 FU_VEC(5) 未使用FPU/Vector 骨架实例化但 issue_valid=0, FU 分配器不应产生这些类型FU 分配测试 (当前不覆盖)
BC-006inst_pkt_t packed struct 位域对齐SV packed struct 从 LSB 到 MSB 按声明顺序排列; 使用 Yosys/Verilator 需验证 bit 序综合后仿真 vs RTL 仿真对比
BC-007VLEN 参数不一致 (pkg 声明 128, M3 规范 256)VLEN=128 为当前实现值; Vector 骨架使用 VLEN 参数化, 修改 pkg 即可扩展VLEN 参数化编译测试

错误类型检测方法恢复策略

6.3 未定义行为声明

  • priv_e 编码 2'b10 未定义: SV 枚举不产生此值, 若 CSR 读出 2'b10 则行为未定义
  • axi4_if awlock/arlock 位宽为 2 (AXI3), 但 SoC 端口使用 1-bit (AXI4): 高位未使用
  • c_ext_expand 对 Quadrant 3 (ci[1:0]=11) 返回 ci[15:0] (32-bit 指令原样传递, 非压缩指令)

6.4 Stall / Flush / Reset 响应

  • FETCH_WIDTH = ISSUE_WIDTH * 2 (pkg 级约束)
  • AXI_STRB_WIDTH = AXI_DATA_WIDTH / 8 (自动计算)
  • BTB_ENTRIES 必须为 2 的幂 (地址索引)

  • Cache 参数: SIZE = WAYS × (SIZE/WAYS), line 大小必须为 2 的幂
