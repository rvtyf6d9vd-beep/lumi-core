4.4 死锁/活锁分析

  • 死锁防护:
    • ST_WAIT_READY 状态持续驱动 mem_valid_out=1, 保证 SRAM 最终响应 mem_ready_in
    • pending_port1_r 独立于 stall_port1 更新 (ERR-029 修复), 防止 Port 1 永久挂起导致 terminal deadlock
    • ST_COMMIT 无条件 1 cycle 后回到 ST_IDLE, 不可停留
  • 活锁防护:
    • 双 LSU 端口严格 Port 0 优先, Port 1 延后 — 保证有限 cycle 内完成
    • Store Buffer 满时 (sb_full=1) 拒绝入队, 反压上游

6.3 未定义行为声明

  • funct3 = 3'b011 / 3'b110 / 3'b111: 非标准 funct3 值, needs_cross_word_split 对 funct3[1:0]=2'b11 返回 1'b0 (不拆分), gen_byte_enable 返回 4'b1111 (全字节使能)。实际行为: 按全字访问, 不触发异常。
  • SB 满时 Store: SB 模式下 Store 被静默跳过 (数据丢失) — 依赖上游反压保护。BYPASS_SB 模式下不存在此问题。
  • funct3[2] 位 (无符号标志): gen_byte_enable / check_alignment / align_store_data 仅使用 funct3[1:0]; sign_extend_load 使用完整 funct3[2:0] 区分符号/零扩展。

  • ISSUE_WIDTH 必须 ≥ 2 (双 LSU 端口需要至少 2 个 slot)
  • SB_DEPTH 为 2 的幂 (当前固定 8), $clog2(SB_DEPTH) = 3 用于 FIFO 指针位宽
  • BYPASS_SB = 1 时 SB 相关逻辑 (入队/出队/commit) 被跳过, store_commit 永远为 0

8.3 关键算法

8.3.1 Store Buffer FIFO 满/空判断

// sb_entry_t: { valid(1), addr(32), data(32), be(4) } = 69 bits/entry
// SB_DEPTH = 8, 指针位宽 = $clog2(8) = 3

sb_empty = (sb_head == sb_tail);
sb_full  = ((sb_tail + 1) == sb_head) ||
           (sb_tail == 7 && sb_head == 0);  // 环形缓冲回绕

// 入队: sb_tail 处写入, sb_tail++
// 出队: sb_head 处清除 valid, sb_head++

8.3.2 双 LSU 端口串行化 (ERR-029)

// 当同 cycle 两条 MEM 指令时:
if (lsu_valid[0] && lsu_valid[1]) begin
    pending_port1_r <= 1;  // 标记 Port 1 待处理
    port0_done_r    <= 1;  // Port 0 先处理
    stall_port1     = 1;   // stall decode 1 cycle
    mem_busy        = 1;   // 反压
end
// 下一 cycle: pending_port1_r=1 → 处理 Port 1 → batch_done

8.3.3 M 级旁路 valid 逻辑 (SA-10)

for each slot i:
  if m_valid[i]:
    if fu_type == FU_MEM && !inst[5]:  // Load
      m_bypass_valid[i] = mem_ready_in   // 仅 SRAM 完成时有效
    elif fu_type == FU_MEM && inst[5]:   // Store
      m_bypass_valid[i] = 0              // 不旁路
    else:                                 // ALU/MUL/DIV/BRANCH
      m_bypass_valid[i] = 1              // 总是有效
