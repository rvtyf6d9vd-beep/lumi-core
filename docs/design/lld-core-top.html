4. 状态机定义

本模块无显式 FSM。流水线控制逻辑分散在各子模块中 (fetch PC FSM, decode DIB FIFO 状态等)。级间寄存器行为由 stall/flush 条件驱动, 见 §2.2 工作模式表。

6.3 未定义行为声明

  • dc_valid && dc_we 同时 dc_ready=0: 请求保持, 不写入 (等待 ready)
  • E1 级多条指令同时异常: 优先级由 slot 编号决定 (slot 0 优先), 实际行为依赖 Writeback 串行化
  • FPU/VRF 端口当前固定接 0 (TODO: 连接 FPU/Vector), 读写 x0 行为

6.4 Stall / Flush / Reset 响应

全局信号响应行为时序
dec_stallF1/F2 冻结; D/I 不发射新指令; DIB FIFO 不弹出同 cycle 生效 (combinational 到 fetch)
e1_mispredict清除 E1 输入寄存器; M 级保留分支指令, 杀死推测指令; fetch 重定向到 e1_br_target下一 clk↑ 生效
trap_request清除 E1 和 M 级输入; fetch 重定向到 trap_pc下一 clk↑ 生效
reset_n异步复位所有流水线寄存器 (valid=0, 数据清零)异步
