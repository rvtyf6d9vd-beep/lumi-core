1. 概述与范围

1.1 模块职责

PMA & AXI 总线模块是 Lumi-Core 内存子系统的权限检查与总线互联核心，负责：

  • PMA 检查器：硬连线的物理内存属性检查，按地址范围判定 cacheable/executable/io 等属性
  • PMA+PMP 联合检查：每次内存访问必须同时通过 PMA (静态) + PMP (动态) 权限验证
  • AXI4 主总线：128bit AXI4 Master 接口，连接外部 DDR/MMIO
  • FFP 外设接口：32bit AXI4 从接口，连接 Form-Factor Peripheral
  • 总线矩阵：多 Master (Core/ICache refill/DCache refill/Trace) 仲裁到 AXI 总线

2.1.2 PMA 属性检查逻辑

// PMA 硬连线检查器
// 每个 PMA 区域用地址比较器 + 属性寄存器硬编码

struct PMA_Entry {
    addr_start : uint32
    addr_end   : uint32
    cacheable      : bool
    executable     : bool
    main_memory    : bool
    io             : bool
    idempotent     : bool
    amo_supported  : bool
    misaligned_ok  : bool
}

// 默认 3 个 PMA 条目 (硬连线)
const PMA_TABLE[3] = {
    {0x0000_0000, 0x0002_FFFF, false, true,  true,  false, true,  false, false},  // TCM
    {0x8000_0000, 0xBFFF_FFFF, true,  true,  true,  false, true,  true,  false},  // DDR
    {0xC000_0000, 0xF000_FFFF, false, false, false, true,  false, false, false},  // MMIO
}

function pma_check(addr, access_type):
    for entry in PMA_TABLE:
        if addr >= entry.addr_start and addr <= entry.addr_end:
            // 命中 PMA 区域, 执行属性检查
            result = PMA_PASS

            if access_type == FETCH and not entry.executable:
                result = PMA_FAULT  // Instruction Access Fault (mcause=1)

            if access_type == AMO and not entry.amo_supported:
                result = PMA_FAULT  // Store/AMO Access Fault (mcause=7)

            if not entry.misaligned_ok and is_misaligned(addr, access_type):
                result = PMA_MISALIGNED  // Address Misaligned (mcause=0/4/6)

            return {entry, result}

    // 未命中任何区域
    return {NULL, PMA_FAULT}  // Access Fault

2.2 PMA+PMP 联合检查 审查约束: PMA+PMP 联合

2.2.1 联合检查架构

每次内存访问必须同时通过 PMA (硬件静态属性) 和 PMP (软件动态权限) 双重检查：

// PMA + PMP 联合检查流程
//
// 内存访问请求 (addr, type, privilege)
//         │
//         ├──→ [PMA 检查] (硬连线, 组合逻辑)
//         │         │
//         │    pma_result = {region_attrs, pass/fault}
//         │
//         ├──→ [PMP 检查] (CSR 动态, 来自 privilege-pmp.html)
//         │         │
//         │    pmp_result = {R/W/X, pass/fault}
//         │
//         ▼
//    [联合判定]
//         │
//    access_allowed = pma_result.pass AND pmp_result.pass
//         │
//    ┌────┼────────────────┐
//    │    │                │
//    ▼    ▼                ▼
//  ALLOW  PMA_FAULT       PMP_FAULT
//         (mcause=1/5/7)  (mcause=1/5/7)
//
// 优先级: PMA 先检查 (组合逻辑更快), PMP 后检查
// 两者同时失败时报告 PMA fault (更底层)

function combined_check(addr, access_type, privilege):
    // Step 1: PMA 检查 (组合逻辑, 0 cycle)
    pma_result = pma_check(addr, access_type)
    if pma_result.result != PMA_PASS:
        raise_access_fault(pma_result, addr)
        return ACCESS_DENIED

    // Step 2: PMP 检查 (CSR 读取 + 比较)
    pmp_result = pmp_check(addr, access_type, privilege)
    if pmp_result != PMP_PASS:
        raise_access_fault(pmp_result, addr)
        return ACCESS_DENIED

    // Step 3: 联合通过 — 返回 PMA 属性供下游使用
    return {ACCESS_ALLOWED, pma_result.region_attrs}

2.2.2 PMA 与 PMP 职责对比

特性PMAPMP
配置方式硬连线 (RTL 固定)CSR 动态 (pmpcfg0~15 + pmpaddr0~63)
变更时机仅在 SoC 集成时修改运行时软件配置
检查内容物理属性 (cacheable/executable/io)访问权限 (R/W/X per privilege)
粒度大区域 (3~6 个)细粒度 (最多 64 个 region)
安全角色物理约束 (不可软件绕过)软件隔离 (M-mode 管理)
检查时序组合逻辑 (0 cycle)1 cycle (CSR 读取 + 比较)
失败结果Access FaultAccess Fault
审查约束 — PMA+PMP 联合:
  • 逻辑关系: AND — 两者必须同时通过。PMA 提供物理属性约束 (如 MMIO 不可执行)，PMP 提供软件权限约束 (如 U-mode 不可访问 M-mode 数据)
  • 时序关系: PMA 先检查 (组合逻辑 0 cycle)，PMP 后检查 (CSR 读取)。PMA fail 时 PMP 检查可跳过 (短路优化)
  • 错误区分: PMA fault 和 PMP fault 均编码为 Access Fault (mcause=1/5/7)，mtval = 失败地址。软件无法区分两者 (设计决策：PMA 为硬件底层约束)
  • 可配 PMA: 高阶客户可通过 cfgaddr/cfgdata CSR 运行时定制 PMA (选项)，此时 PMA 变为半动态，但默认仍为硬连线
  • 验证策略: Cover Group cg_pma_pmp_combined 覆盖 PMA×PMP 全组合 (pass/pass, pass/fail, fail/pass, fail/fail)

2.2.3 PMA 属性对下游模块的影响

// PMA 属性传递给 cache/tcm 模块
function route_with_pma(addr, access_type):
    {status, pma_attrs} = combined_check(addr, access_type, current_privilege)
    if status == ACCESS_DENIED:
        return ACCESS_FAULT

    // 根据 PMA 属性决定路由
    if pma_attrs.io:
        // MMIO: 不缓存, 直接 AXI
        return route_axi_direct(addr, access_type)

    elif not pma_attrs.cacheable:
        // TCM 区域 (cacheable=false): 走 TCM 路径
        return route_tcm(addr, access_type)

    else:
        // DDR (cacheable=true): 走 Cache 路径
        return route_cache(addr, access_type, pma_attrs)

    elif nc_req.valid:

2.5.2 位宽转换逻辑

// 128bit → 32bit 写数据拆分
function ffp_write_bridge(addr, data128, strb16):
    // 拆分为 4 次 32bit AXI burst
    for i in 0..3:
        ffp_wdata = data128[i*32+31 : i*32]
        ffp_wstrb = strb16[i*4+3 : i*4]
        ffp_addr  = addr + i*4
        axi_ffp_write(ffp_addr, ffp_wdata, ffp_wstrb)

// 32bit → 128bit 读数据拼接
function ffp_read_bridge(addr):
    for i in 0..3:
        ffp_rdata[i*32+31 : i*32] = axi_ffp_read(addr + i*4)
    return ffp_rdata  // 128bit

4.2 AXI 总线仲裁状态机

// AXI 总线仲裁 FSM
enum ArbState { IDLE, GRANT_DC, GRANT_IC, GRANT_NC, GRANT_TRACE, WAIT_RESP }

function arb_fsm():
    state = IDLE
    while true:
        if dc_req.valid:
            state = GRANT_DC
            axi_connect(dc_master)
        elif ic_req.valid:
            state = GRANT_IC
            axi_connect(ic_master)
        elif nc_req.valid:
            state = GRANT_NC
            axi_connect(nc_master)
        elif trace_req.valid:
            state = GRANT_TRACE
            axi_connect(trace_master)
        else:
            state = IDLE

        if state != IDLE:
            // 等待当前 burst 完成
            wait_until(axi_burst_complete)
            state = IDLE
