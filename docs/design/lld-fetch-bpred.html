Branch redirect 边沿检测 (SA-CM-REDIRECT-FIX): branch_redirect_valid 是短脉冲信号, FSM override 仅在 branch_redirect_valid && !branch_redirect_valid_d (上升沿) 时触发, 且 state_reg != ST_FLUSH 时生效。flush 期间的 redirect 被忽略, 防止错误路径指令反复覆盖 PC。

4.4 死锁/活锁分析

  • 死锁防护: ST_FLUSH 使用 flush_cnt 倒计时 (初始值 2), 保证有限步内退出。ST_STALL 在 redirect/trap 时也能跳出, 不会永久 stall。
  • 活锁防护: default 状态转移回 ST_IDLE (ERR-057), 防止 FSM 进入非法状态后死循环。Debug halt 为电平触发, 外部释放后自动恢复。
  • Post-flush hold (SA-CM-PCFIX): post_flush_hold_r 保证 flush 后第一个 ST_FETCH 周期 PC 不前进, 使 predecode 用正确的 start_offset 处理 F2 捕获的数据。

BC-006 F2 重复捕获导致 DIB 重复写入: stall 时 PC 不前进, 相同 F2 数据被反复送入 DIB !dec_stall 条件门控 F2 捕获; stall 时 f2_valid_r ← 0 L512: !dec_stall 条件; L525: valid 清除

6.3 未定义行为声明

  • branch_redirect_valid 和 trap_redirect_valid 同时为 1: 行为已定义 — trap 优先 (SA-CM-007)
  • BTB 索引回绕: grp_base_idx + p 可能超过 BTB_ENTRIES, 行为依赖 SRAM 地址回绕 (综合工具处理)
  • RAS 为空时 peek: ras_top == 0 时读取 ras_stack[RAS_DEPTH-1], 若从未 push 则返回 0
  • debug_halt 在 ST_FLUSH 期间拉高: flush 完成后进入 ST_FETCH, 然后检测到 halt 进入 ST_HALT

6.4 Stall / Flush / Reset 响应

全局信号响应行为时序
dec_stallFSM 保持 PC 不变; f2_valid_r ← 0 防止 DIB 重复写入; carry 保持同 cycle FSM 判断; F2 寄存器下一 clk↑ 生效
dib_not_full = 0同 dec_stall: PC hold, F2 数据保持同 cycle FSM 判断
branch_redirect_valid组合清除 f2_valid; 进入 ST_FLUSH (2 cycle); PC 设为 redirect PC; carry 清零组合立即清除 f2_valid; 状态/PC 下一 clk↑ 更新
trap_redirect_valid同 branch redirect, 但优先级更高, PC 设为 mtvec同上
reset_n = 0异步复位所有寄存器: FSM→ST_IDLE, PC→RESET_VECTOR, BTB/RAS/LTAGE 全部清零异步

8.3 关键数据结构

BTB 条目 (btb_entry_t)

typedef struct packed {
    logic                  valid;        // 1-bit: 条目有效
    logic [BTB_TAG_W-1:0]  tag;          // 18-bit: PC 高位 tag
    logic [31:0]           target;       // 32-bit: 分支目标地址
    logic                  is_call;      // JAL/JALR with rd=x1/x5
    logic                  is_ret;       // JALR with rs1=x1/x5, rd=x0
    logic                  is_branch;    // conditional branch
} btb_entry_t;                          // 总 54 bit

LTAGE 条目 (ltage_entry_t)

typedef struct packed {
    logic                    valid;      // 1-bit: 条目有效
    logic [7:0]              tag;        // 8-bit: PC XOR history tag
    logic [LTAGE_CTR_W-1:0]  ctr;        // 3-bit: 饱和计数器 (MSB=预测方向)
    logic [LTAGE_U_W-1:0]    useful;     // 2-bit: useful 计数器
} ltage_entry_t;                         // 总 14 bit

8.4 关键算法

LTAGE 几何历史长度计算 (generate)

// 每表的历史长度 (几何递增):
// t=0: 4, t=1: 6, t=2: 9, t=3: 14
// t=4: 21, t=5: 33, t=6: 45, t=7: 57
// t=8: 72, t=9: 112, t=10: 152, t=11: 192
// 截断到 LTAGE_HIST_MAX (64)

// 索引 = PC[10:1] XOR branch_history[ACTUAL_HIST-1 : ACTUAL_HIST-10]
// Tag  = PC[15:8] XOR branch_history[ACTUAL_HIST-1 : ACTUAL_HIST-8]

分支预测决策优先级

1. Slot-0 BTB hit (pc_reg 处):
   - RET → RAS peek, ras_pop
   - CALL + branch → LTAGE 预测, ras_push
   - CALL + !branch → always taken, ras_push
   - branch → LTAGE 预测
   - other → always taken

2. 组级 BTB hit (group 内 PC 之后):
   覆盖 slot-0 结果, 同样的类型判断逻辑
   pred_branch_slot = grp_slot (用于 predecode 截断)

3. 无 BTB hit → pred_taken=0, target=pc_reg+4

Predecode 扫描算法

1. 构造 eff_data (144-bit):
   - carry_valid: {raw_swapped[127:0], carry_hw[15:0]}, bo=2
   - 无 carry: {16'h0, raw_swapped[127:0]}, bo=start_offset

2. 主循环 (最多 FETCH_WIDTH=6 条):
   - 检查 pred_taken 截断 (i > pred_branch_slot → stop)
   - 读取 hw = eff_data[bo*8 +: 16]
   - hw[1:0] != 2'b11 → 压缩指令, c_ext_expand, bo+=2
   - hw[1:0] == 2'b11 → 32-bit 指令, 检查边界
     - 跨边界 → carry_hw_out = hw, stop
     - 未跨 → 读取 32-bit, bo+=4

3. bytes_consumed:
   - carry_valid 或 carry_valid_out → 16 - start_offset (跳到下一对齐块)
   - 否则 → bo - start_offset
