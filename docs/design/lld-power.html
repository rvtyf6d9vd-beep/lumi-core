4. 状态机定义

4.1 主 FSM — pwr_state_e (3-bit, 5 states)

状态名编码含义进入条件
ST_RUN3'd0正常运行reset / wakeup_done
ST_IDLE3'd1ICG 门控, 等待唤醒wfi_req + !core_active
ST_SLEEP3'd2深层低功耗idle_cnt ≥ 4 in IDLE
ST_DEEP_SLEEP3'd3PLL 关闭!vcc_on (deep_sleep_req)
ST_WAKEUP3'd4唤醒序列 (≤8 cycle)wake_event in IDLE/SLEEP/DEEP

4.2 状态转移图

stateDiagram-v2
    [*] --> ST_RUN : reset
    ST_RUN --> ST_IDLE : wfi_req && !core_active
    ST_RUN --> ST_IDLE : wrs_req && reservation_valid
    ST_RUN --> ST_DEEP_SLEEP : !vcc_on
    ST_IDLE --> ST_WAKEUP : wake_event
    ST_IDLE --> ST_SLEEP : idle_cnt >= 4
    ST_IDLE --> ST_DEEP_SLEEP : !vcc_on
    ST_SLEEP --> ST_WAKEUP : wake_event
    ST_DEEP_SLEEP --> ST_WAKEUP : vcc_on && wake_event
    ST_WAKEUP --> ST_RUN : wakeup_done (cnt>=8)

4.3 状态转移条件表

源状态目标状态触发条件输出动作
ST_RUNST_IDLEwfi_req && !core_activeicg_en=1 (下一 cycle)
ST_RUNST_IDLEwrs_req && reservation_valid && !core_activeicg_en=1
ST_RUNST_DEEP_SLEEP!vcc_oncore_sleep=1, icg_en=1
ST_IDLEST_WAKEUPcore_active || irq_pending || reservation_invalidate || !sys_reset_nicg_en=0, wakeup_cnt 开始
ST_IDLEST_SLEEPidle_cnt ≥ 4 && !wake_eventcore_sleep=1
ST_IDLEST_DEEP_SLEEP!vcc_oncore_sleep=1
ST_SLEEPST_WAKEUPwake_eventicg_en=0
ST_DEEP_SLEEPST_WAKEUPvcc_on && wake_eventicg_en=0
ST_WAKEUPST_RUNwakeup_cnt ≥ 8icg_en=0, core_sleep=0

4.4 死锁/活锁分析

  • 死锁防护 (WAKEUP): wakeup_cnt 无条件递增, wakeup_done 在 8 cycle 内必然到达
  • 死锁防护 (DEEP_SLEEP): 唤醒需要 vcc_on + wake_event 双条件; 若 SoC 不恢复 vcc_on, 则永远在 DEEP_SLEEP (预期行为)
  • 活锁防护 (IDLE): wake_event 包含 irq_pending, 中断必然唤醒; !sys_reset_n 也可唤醒 (复位退出)
