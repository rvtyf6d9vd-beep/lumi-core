4. 状态机定义

4.1 Lockstep 主 FSM — ls_state_e (2-bit, 4 states)

状态名编码含义进入条件
ST_IDLE2'd0锁步未使能, 不比较reset / lockstep_enable=0
ST_COMPARE2'd1持续比较 W 级输出lockstep_enable=1
ST_MISMATCH2'd2检测到不一致, 锁存错误any_mismatch in ST_COMPARE
ST_RECOVER2'd3等待外部恢复 (NMI/Reset 处理)从 ST_MISMATCH 自动进入

4.2 状态转移图

stateDiagram-v2
    [*] --> ST_IDLE : reset
    ST_IDLE --> ST_COMPARE : lockstep_enable
    ST_COMPARE --> ST_IDLE : !lockstep_enable
    ST_COMPARE --> ST_MISMATCH : any_mismatch
    ST_MISMATCH --> ST_RECOVER : 自动 (下一 cycle)
    ST_RECOVER --> ST_IDLE : !lockstep_enable
    ST_RECOVER --> ST_COMPARE : reset_n && !any_mismatch

4.3 状态转移条件表

源状态目标状态触发条件输出动作
ST_IDLEST_COMPARElockstep_enable == 1开始逐 cycle 比较
ST_COMPAREST_IDLElockstep_enable == 0停止比较
ST_COMPAREST_MISMATCHany_mismatch == 1锁存 error_pc/error_slot, lockstep_error=1
ST_MISMATCHST_RECOVER无条件 (1 cycle)按 mode 输出 NMI/Reset/Log
ST_RECOVERST_IDLElockstep_enable == 0清除错误, 回到空闲
ST_RECOVERST_COMPAREreset_n && !any_mismatch重新同步, 继续比较

4.4 死锁/活锁分析

  • 死锁防护 (MISMATCH→RECOVER): ST_MISMATCH 仅持续 1 cycle, 无条件转移到 ST_RECOVER
  • 死锁风险 (RECOVER): 若 NMI 处理失败且 lockstep_enable 保持 1, 同时 mismatch 条件持续, 则永远停留在 ST_RECOVER。预期行为 — 需 SoC 级看门狗兜底
  • 活锁防护: ST_COMPARE 不消耗资源, 纯组合比较; 无循环等待

5. 数据通路与时序

5.1 Round-Robin AXI 仲裁流程

graph LR
    REQ["arb_req[i] = hart_dc_valid[i] || hart_ic_req[i]"] --> RR["Round-Robin 选择器"]
    RR --> SEL["arb_sel (已锁定胜出 Hart)"]
    SEL --> GRANT["arb_grant = 1"]
    GRANT --> MUX["axi_araddr = hart_dc_addr[arb_sel]"]
    GRANT --> RDY["hart_dc_ready[i] = (arb_sel == i)"]

5.2 仲裁 cycle 分解

Cyclearb_selarb_reqarb_grantaxi_arvalid备注
Nkreq[k]=111Hart k 获得总线
N+1(k+1)%Nreq[(k+1)%N]=111轮转到下一 Hart
N+1不变无请求00无请求时 arb_grant=0

5.3 Lockstep 比较数据通路

graph LR
    A["Hart A: commit_valid/pc/result"] --> CMP["逐 slot 比较 (IW 参数化)"]
    B["Hart B: commit_valid/pc/result"] --> CMP
    CMP --> MM["valid_mismatch / pc_mismatch / result_mismatch"]
    MM --> EC["error_code[3:0] 编码"]
    MM --> FIRST["首个 mismatch slot 锁存"]
    EC --> LOG["lockstep_log = {PC, code, timestamp}"]
    FIRST --> ERRPC["error_pc = commit_pc_a[mismatch_slot]"]

5.4 Lockstep 比较时序

信号类型延迟说明
valid_mismatch[i]combinational0 cycleXOR 比较
pc_mismatch[i]combinational0 cycle仅双 valid 时比较
result_mismatch[i]combinational0 cycle仅双 valid 时比较
any_mismatchcombinational0 cycle所有 mismatch OR 聚合
error_pcregistered1 cycleST_COMPARE→ST_MISMATCH 时锁存

5.5 JTAG TAP Chain 连接

tdi → jtag_tdo_chain[0] → Hart_0 → chain[1] → Hart_1 → ... → chain[NUM_HARTS] = tdo

当前 RTL 为 bypass 链 (jtag_tdo_chain[i+1] = jtag_tdo_chain[i]), 实际需替换为各 Hart Debug Module 的 TAP。

7.1 参数约束关系

  • NUM_HARTS=1 时 arb_sel 位宽 = $clog2(1) = 0, Round-Robin 退化为直连
  • IW 决定 error_code 位宽 (固定 4-bit, [3] 预留) 和 mismatch_slot 位宽 ($clog2(IW))
  • error_slot 固定 4-bit, 高 2-bit 补零: {2'h0, mismatch_slot}
