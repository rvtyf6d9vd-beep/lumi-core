4.4 死锁/活锁分析

  • 死锁防护: 所有 FSM 状态均在 1~2 cycle 内回到 ST_IDLE; default 分支强制跳转 ST_IDLE, 防止非法状态锁死。
  • 活锁防护: 核心优先仲裁 — DMA 请求仅在核心空闲 (!core_valid || !any_hit) 时受理, 避免 DMA 持续抢占。但核心连续访问 TCM 会导致 DMA 饥饿; 软件层面应通过 burst 间隔释放总线。
  • DMA pending 清理: dma_pending 在 ST_DMA_WRITE 或 ST_ECC_CHECK (DMA 读路径) 时自动清零, 防止重复访问。

6. 边界条件与异常处理

6.1 边界条件清单

编号条件处理策略验证覆盖
BC-001 地址越界: core_addr 不在 ITCM/DTCM0/DTCM1 任何区域 any_hit=0, core_hit=0; FSM 不离开 ST_IDLE, SRAM 不被访问; core_ready=1 (空转) 遍历间隙地址: 0x0000_8000, 0x0001_8000, 0x0002_8000, 0xFFFF_FFFF
BC-002 bank_offset 最大边界: 访问各区域最后一个字 (offset = WORDS_PER_BANK - 1 = 8191) bank_offset = addr[14:2], 32KB 时 addr[14:2] 最大 13'h1FFF = 8191, 正好覆盖 ITCM/DTCM0/DTCM1 各末地址读写
BC-003 ECC 双 bit 错误 (DED): 存储数据发生 2-bit 翻转 syndrome!=0 && overall_parity==0 → ded_flag=1; ecc_ded_irq 脉冲; 数据不纠正直接输出 (不可靠); 软件需处理 NMI 注入所有 C(39,2)=741 种双 bit 组合的代表性子集
BC-004 ECC 使能关闭: ECC_EN=0 时的读写行为 写路径不调用 ecc_encode(); 读路径 ecc_ce_flag/ecc_ded_flag 恒 0; ecc_ce_irq/ecc_ded_irq 恒 0; core_rdata = read_data_raw ECC_EN=0 参数化仿真
BC-005 DMA 与核心同 cycle 请求: core_valid=1 && dma_arvalid=1 同时到达 ST_IDLE FSM 优先级: core_valid && any_hit 检查在前, DMA 检查在后; 核心获得总线, DMA 请求保持 pending 直到核心释放 同 cycle 注入 core+DMA 请求, 验证核心优先
BC-006 DMA 读时 dma_rready=0: DMA master 未及时接收读数据 dma_rvalid_reg 保持为 1, 直到 dma_rready=1 时清零; 数据保持 dma_rdata_reg 不变 DMA rready 延迟 1~10 cycle 后拉高
BC-007 ECC syndrome 指向 bit 33 (ecc[5] 自身): Hamming 位错误 RTL 处理: syndrome > 6'd32 且 syndrome == 6'd33 时翻转 corrected[31]; syndrome > 33 时不纠正 (超出范围, 视为 DED) 注入 ecc[5:0] 各位单 bit 翻转
BC-008 复位后立即访问: reset_n 释放后第一 cycle 发 core_valid FSM 复位值 ST_IDLE, core_ready=1; 所有寄存器清零; SRAM 内容不确定 (综合后为 0 或 X) reset → 1 cycle → read, 验证 FSM 正确响应

6.2 错误检测与恢复

错误类型检测方法恢复策略
单 bit 数据翻转 (CE)ECC syndrome != 0, overall_parity = 1自动纠正 + 回写修正值 + ecc_ce_cnt++ + ecc_ce_irq 脉冲
双 bit 数据翻转 (DED)ECC syndrome != 0, overall_parity = 0ecc_ded_irq NMI 中断; 数据不可纠正; 软件负责处理 (bus error exception)
ECC 位自身单 bit 翻转syndrome = 0, overall_parity = 1RTL 注释标注 "忽略" (不影响数据正确性)
非法 FSM 状态default 分支强制跳转 ST_IDLE

6.3 未定义行为声明

  • 部分写 + ECC 读出: core_be != 4'hF 写入后 ECC 校验位过期, 后续全字读出 ECC 解码结果未定义 (可能误报 CE/DED)。
  • core_valid 持续拉高: 若 core_valid 在 ST_CORE_READ/ST_CORE_WRITE 期间持续为 1, FSM 回到 ST_IDLE 后会立即重新进入, 相当于连续访问 — 这是合法行为但需确保地址/数据同步更新。
  • DMA burst: 当前 RTL 不实现 AXI burst 计数 (无 awlen 端口), 每次 DMA 操作为单字 (single transfer)。HLD 中描述的 INCR burst 需在外部 DMA 控制器侧管理。

6.4 Stall / Flush / Reset 响应

7.1 参数约束关系

  • ITCM_SIZE 和 DTCM_SIZE 必须为 4 的倍数 (字对齐), 且为 2 的幂。
  • WORDS_PER_BANK = ITCM_SIZE / 4, 当前固定 8192 (32KB/4); bank_offset 位宽 = log2(WORDS_PER_BANK) = 13 bit。
  • 地址解码使用 addr[14:2] 作为 bank_offset, 隐含假设各区域 ≤ 32KB (addr[14:0] = 15 bit = 32KB); 若区域 > 32KB 需调整位域。
  • 三区域总容量 = ITCM_SIZE + 2 * DTCM_SIZE, 默认 96KB。
  • LATENCY_MODE 为 elaboration-time 参数 (非运行时 CSR); HLD 描述的运行时切换需额外包装逻辑。

8.3 关键算法

8.3.1 ECC 编码 (Hamming SECDED)

// ecc_encode: 32-bit data → 7-bit ECC
// ecc[5:0] = 6 个 Hamming 奇偶位, 每个覆盖特定数据位子集
// ecc[6]   = 整体奇偶位 (SECDED 扩展)
//
// ecc[0] covers: data bits with bit0=1 in position index
// ecc[1] covers: data bits with bit1=1 in position index
// ecc[2] covers: data bits with bit2=1 in position index
// ecc[3] covers: data bits with bit3=1 in position index
// ecc[4] covers: data bits with bit4=1 in position index
// ecc[5] covers: data[31]
// ecc[6] = parity(data[31:0] ^ ecc[5:0])

8.3.2 ECC 解码与纠正

// ecc_decode_fn: (data[31:0], stored_ecc[6:0]) → {ded, ce, corrected[31:0]}
//
// 1. 重算 syndrome[5:0] = stored_ecc[5:0] XOR recomputed_hamming(data)
// 2. 重算 overall_parity = stored_ecc[6] XOR parity(data ++ stored_ecc[5:0])
// 3. 判定:
//    syndrome=0, parity=0 → 无错误
//    syndrome=0, parity=1 → ECC 位自身错误 (忽略)
//    syndrome≠0, parity=1 → 单 bit 错误: syndrome 值 = 错误位位置 (1~33)
//                            syndrome ≤ 32: corrected[syndrome-1] ^= 1
//                            syndrome = 33: corrected[31] ^= 1
//    syndrome≠0, parity=0 → 双 bit 错误: 不可纠正, ded_flag=1

8.3.3 地址解码

// 地址解码逻辑 (active_addr 选择)
// if FSM 处于 DMA 状态: active_addr = dma_addr_reg
// else:                  active_addr = core_addr
//
// itcm_hit  = (active_addr >= 0x0000_0000) && (active_addr < 0x0000_0000 + ITCM_SIZE)
// dtcm0_hit = (active_addr >= 0x0001_0000) && (active_addr < 0x0001_0000 + DTCM_SIZE)
// dtcm1_hit = (active_addr >= 0x0002_0000) && (active_addr < 0x0002_0000 + DTCM_SIZE)
//
// bank_offset = active_addr[14:2]  // 字偏移, 13 bit for 8192 words (32KB)
