1. 概述与范围

1.1 模块职责

Decode & Issue 模块覆盖 Lumi-Core 8 级流水线的 D (Decode) 和 I (Issue) 两级：

  • D 级：完整指令解码、寄存器读取 (从寄存器堆)、操作数准备
  • I 级：指令发射到 FU、数据旁路 (Forwarding)、发射仲裁 (含 skip-stalled)
GEM5 映射: Spec 的 D 级对应 GEM5 的 Decode1 (预解码) + Decode2 (完整解码+寄存器读)；Spec 的 I 级对应 GEM5 的 Issue 级。

1.2 范围边界

包含不包含
完整 RV32 指令解码 (含 M/A/F/D/C/B/V/Zce 扩展)指令取指 (属于 fetch-bpred.html)
三发射仲裁与 skip-stalled 机制FU 执行 (属于 execute-alu.html)
数据旁路网络 (5 路: E1/E2/M/W/FPU→D)寄存器堆存储阵列 (属于 regfile.html)
冒险检测与 stall 逻辑旁路数据来源 (由各级 FU 输出)

关联模块: fetch-bpred.html (取指结果) | execute-alu.html (FU 分配) | regfile.html (读端口仲裁) | writeback-csr.html (CSR 指令) | fpu-vector.html (FPU/Vector FU 分配)

2. 功能描述

2.1 D 级：指令解码

2.1.1 解码逻辑

字段位域解码输出
opcode[6:0]inst[6:0]指令大类 (OP/OP-IMM/LOAD/STORE/BRANCH/SYSTEM 等)
rd[4:0]inst[11:7]目标寄存器号
funct3[2:0]inst[14:12]子操作类型
rs1[4:0]inst[19:15]源寄存器 1 号
rs2[4:0]inst[24:20]源寄存器 2 号
funct7[6:0]inst[31:25]扩展子操作 (MULDIV/FPU/AMO 等)
imm按格式提取立即数 (I/S/B/U/J 格式)

2.1.2 寄存器读取

D 级同时从寄存器堆读取操作数，每个 cycle 最多读 3 条指令的源寄存器：

发射槽读端口需求描述
Slot 0rs1 + rs2 (2R)第一条指令
Slot 1rs1 + rs2 (2R)第二条指令
Slot 2rs1 + rs2 (2R)第三条指令 (可能 stall)
D-009 端口压力 D-009: 3 条指令 × 2 源 = 6 读请求，但整数寄存器堆仅 3R 端口。最坏情况需分 2 cycle 完成。仲裁规则: 程序顺序优先 (Slot 0 > Slot 1 > Slot 2)，端口不足时 Slot 2 stall 1 cycle。

2.1.3 寄存器读端口仲裁伪代码 D-009

// 整数寄存器堆 3R/2W 端口仲裁
function regfile_read_arbitrate(slot0, slot1, slot2):
    read_ports_available = 3  // 3R ports
    granted = []

    // 按程序顺序依次授权
    for slot in [slot0, slot1, slot2]:
        needed_ports = count_unique_rs(slot)  // 1 or 2
        if needed_ports <= read_ports_available:
            granted.append(slot)
            read_ports_available -= needed_ports
        else:
            stall(slot)  // 端口不足, stall 该指令
            break

    return granted

2.2 I 级：指令发射

2.2.1 发射仲裁

每条解码后的指令根据类型分配到对应 FU：

指令类型目标 FU可用数量
ALU (add/sub/and/or/xor/slt/shift)LumiIntFU4
MUL (mul/mulh/mulhsu/mulhu)LumiIntMulFU1
DIV (div/divu/rem/remu)MinorDefaultIntDivFU1
LOAD/STORE (lb/lh/lw/sb/sh/sw/AMO)MinorDefaultMemFU2
FPU (fadd/fmul/fdiv/fsqrt/fmadd...)MinorDefaultFloatSimdFU1
SYSTEM (ECALL/EBREAK/FENCE/SFENCE)MinorDefaultMiscFU1
BRANCH (beq/bne/blt/bge/bltu/bgeu)LumiIntFU (复用)4

2.2.2 Skip-Stalled 机制 D-015

skip-stalled 不破坏程序顺序 (in-order commit): 被跳过的指令仍在发射队列中等待，后续指令仅在同一 FU 类型不冲突时才越过发射，所有指令仍按程序顺序提交 (W 级)。
// Skip-Stalled 发射伪代码
function issue_arbitrate(inst_queue[0..2]):
    for i in range(3):
        inst = inst_queue[i]
        target_fu = classify(inst)

        if fu_available(target_fu):
            issue(inst, target_fu)
            mark_fu_busy(target_fu)
        elif skip_stalled_enabled:
            // 跳过该指令, 检查后续指令是否可以使用不同 FU
            mark_inst_stalled(inst)
            continue  // 检查下一条
        else:
            // 不启用 skip-stalled, 整体 stall
            stall_pipeline()
            break

2.2.3 数据旁路网络

I 级实现 5 路数据旁路 (Forwarding/Bypass)，消除大部分 RAW 冒险：

旁路路径来源级目标数据
E1→DExecute 1 级Decode 级寄存器读ALU/Branch 运算结果
E2→DExecute 2 级Decode 级寄存器读MUL 中间/最终结果
M→DMemory 级Decode 级寄存器读Load 返回数据
W→DWriteback 级Decode 级寄存器读最终写回数据
FPU→DFPU 结果Decode 级寄存器读FPU 运算结果
D-017 RTL 关键路径约束: 5 路旁路 mux (5→1, 32bit = 160bit) 延迟约 80~100ps，占 I 级 125ps 预算的 64~80%。RTL 骨架中预留流水化分割点。

2.3 冒险处理

冒险类型检测方式处理策略代价
RAW (旁路命中)rs1/rs2 与前序指令 rd 匹配旁路转发0 cycle
Load-Use RAW前序为 Load + rs 匹配1 stall (推测命中时 0)0~1 cycle
FU 冲突同类型 FU 超额请求skip-stalled / stall0~N cycle
DIV 占用DIV FU busy (9 cycle)其他 FU 继续, DIV FU stall0 (其他 FU)
WAW/WAR顺序执行天然消除无需处理0

2.4 推测性 Load 延迟预测

GEM5 参数 executeLoadAssumedLatency=1：Load 指令在 D 级推测 L1D Cache 命中 (延迟 1 cycle)，后续指令不 stall。若实际 Cache miss，在 M 级检测并回滚 (flush + stall)。

4. 数据通路

4.1 D 级到 I 级数据流

F2 输出 ──→ D级解码 ──→ 寄存器读取 ──→ 操作数准备 ──→ I级发射仲裁 ──→ FU
                │              │                              │
                ├─ inst_type   ├─ rs1_data                    ├─ bypass_mux
                ├─ rd/rs1/rs2  ├─ rs2_data (或旁路)           ├─ fu_select
                ├─ imm         └─ (3R端口仲裁)               └─ skip_stalled
                └─ pred_info

4.2 旁路 mux 选择逻辑

// 每个源操作数的旁路选择 (5→1 mux)
function bypass_select(rs_addr, rs_regfile_data):
    if (e1_valid && e1_rd == rs_addr):
        return e1_result           // E1→D 旁路 (最高优先级)
    elif (e2_valid && e2_rd == rs_addr):
        return e2_result           // E2→D 旁路
    elif (m_valid && m_rd == rs_addr):
        return m_result            // M→D 旁路
    elif (w_valid && w_rd == rs_addr):
        return w_result            // W→D 旁路
    elif (fpu_valid && fpu_rd == rs_addr):
        return fpu_result          // FPU→D 旁路
    else:
        return rs_regfile_data     // 无旁路, 使用寄存器堆读出值
