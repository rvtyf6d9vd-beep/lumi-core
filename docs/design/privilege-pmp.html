1. 概述与范围

1.1 模块职责

特权架构 & PMP 模块实现 Lumi-Core 的三级特权模式和物理内存保护：

  • 特权模式管理：M-mode (Machine) / S-mode (Supervisor) / U-mode (User) 三级特权
  • 特权切换：Trap 进入高特权级 / xRET 返回低特权级
  • PMP 检查器：8 个 Region (可配) 的物理内存访问权限检查
  • Smepmp (ePMP)：增强型 PMP，通过 mseccfg CSR 锁定 M-mode 权限
  • CSR 访问控制：按特权级限制 CSR 可读/可写范围

2.2.2 PMP 匹配模式

// PMP 匹配模式
enum PMP_MatchMode {
    OFF  = 0,  // 禁用
    TOR  = 1,  // Top of Range: [pmpaddr[i-1], pmpaddr[i])
    NA4  = 2,  // Naturally aligned 4-byte: [pmpaddr[i], pmpaddr[i]+4)
    NAPOT= 3   // Naturally aligned power-of-two: 由 pmpaddr 编码确定大小
}

// NAPOT 地址编码
// pmpaddr = (base_addr >> 2) | ((size >> 3) - 1)
// 例如: 16KB region at 0x8000_0000:
//   pmpaddr = (0x8000_0000 >> 2) | ((16384 >> 3) - 1)
//           = 0x2000_0000 | 0x7FF = 0x2000_07FF

// PMP 检查伪代码
function pmp_check(addr, access_type, privilege):
    if privilege == M_MODE and not smepmp_active:
        // M-mode 默认不受 PMP 限制 (Smepmp 未激活时)
        return PMP_PASS

    for i in 0..PMP_NUM_REGIONS-1:
        cfg = pmpcfg[i]
        mode = cfg.A  // 匹配模式

        if mode == OFF:
            continue

        // 地址匹配
        if match(addr, i, mode):
            // 检查权限
            if privilege == M_MODE:
                // M-mode: 仅检查 L=1 的 region
                if cfg.L:
                    return check_rwX(cfg, access_type)
                else:
                    return PMP_PASS  // M-mode 未锁定 region 不限制
            else:
                // S/U-mode: 所有 region 均检查
                return check_rwx(cfg, access_type)

    // 无匹配 region
    if privilege == M_MODE:
        if mmwp_enabled:  // MMWP: M-mode 默认拒绝
            return PMP_FAIL
        else:
            return PMP_PASS  // 默认允许
    else:
        return PMP_FAIL  // S/U-mode 无匹配 = 拒绝

function match(addr, idx, mode):
    case mode:
        TOR:  return addr >= (pmpaddr[idx-1] << 2) and addr < (pmpaddr[idx] << 2)
        NA4:  return (addr & ~0x3) == (pmpaddr[idx] << 2)
        NAPOT: return napot_match(addr, pmpaddr[idx])

2.3 Smepmp (ePMP) ERR-004: MML+MMWP 边界

2.3.1 mseccfg CSR (0x747)

位名称描述
0MMLMachine Mode Lockdown: 激活 M-mode PMP 限制 + 锁定机制增强
1MMWPMachine Mode Whitelist Policy: M-mode 未匹配 region 默认拒绝
2RLBRule Locking Bypass: 允许修改已锁定规则 (调试用)

2.3.2 MML+MMWP 组合行为

MMLMMWPM-mode 行为S/U-mode 行为
00不受 PMP 限制 (传统行为)受 PMP 限制, 无匹配=拒绝
01仅 L=0 region 不限, L=1 限制; 无匹配=拒绝同左
10M-mode 受 PMP 限制, L 位语义变化同左, 但 M-mode 共享 region 受限
11严格白名单: M-mode 仅访问显式授权 region同左
审查约束 — ERR-004 (Smepmp MML+MMWP 边界):
  • 已知问题: Smepmp v1.0 规范中 MML=1 时权限编码表与 MML=0 不同，容易混淆
  • 实现策略: 使用查找表 (LUT) 实现权限编码，而非组合逻辑，避免编码错误
  • MML=1 权限表: L=0 时 R/W/RW 的 M-mode/S-mode 权限含义与 MML=0 时完全不同
  • RLB 安全: RLB=1 时允许修改锁定规则，仅限调试模式。生产环境中 RLB 必须为 0
  • 验证策略: Cover Group cg_smepmp_modes 覆盖 MML×MMWP×RLB 全 8 种组合

2.4 CSR 访问控制

// CSR 访问控制逻辑
function csr_access_check(csr_addr, access_type, privilege):
    // CSR 地址编码:
    // [11:10] = 读写权限 (00=RW, 01=RW, 10=RO, 11=RO/custom)
    // [9:8]   = 最低特权级 (00=U, 01=S, 10=Reserved, 11=M)

    csr_rw   = csr_addr[11:10]
    csr_prv  = csr_addr[9:8]

    // 特权级检查
    if privilege < csr_prv:
        return CSR_ILLEGAL  // 特权级不足

    // 读写权限检查
    if access_type == WRITE and csr_rw[1:0] == 2'b11:
        return CSR_ILLEGAL  // 只读 CSR

    return CSR_OK
