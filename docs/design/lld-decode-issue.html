4.4 死锁/活锁分析

  • 死锁防护: flush 信号可无条件打断 ST_STALL 状态 (flush 优先级 > stall), 避免 FU 永久 busy 导致的死锁。ERR-057 防御性 default 确保非法编码回到 ST_IDLE。
  • 活锁防护: post_flush_block_r 在 DIB 收到新数据 (dib_count > 0) 后自动清零, 不会无限阻塞。DIB dib_not_full 采用保守阈值 (≤ DIB_DEPTH - FETCH_WIDTH), 确保总有空间接纳新 batch。
  • ERR-019 保护: 分支指令 (FU_BRANCH) 发射后 stop_issue=1, 防止推测执行带来的活锁风险。

5.1.1 DIB 数据流细节

DIB (Decoded Instruction Buffer) 是一个 32-entry 环形 FIFO, 解耦 Fetch 与 Issue:

  • 写入: f2_valid=1 时, 将 pd_inst_r 中的有效条目 (≤ FETCH_WIDTH) 写入 dib[dib_wr_ptr + i], 更新 dib_wr_ptr += dib_wr_offset
  • 读取: 组合逻辑从 dib[dib_rd_ptr + s] 读出 FETCH_WIDTH 条, 含 write-through bypass (同周期写入位置直接转发)
  • 计数: dib_count 6-bit (0~32), 每周期原子更新: +dib_wr_offset(f2_valid) - issue_count
  • Flush: flush=1 时三个指针全部清零

5.1.2 D 级解码数据流

从 DIB 读出的指令经过以下处理链:

  1. 压缩指令展开: 若 inst[1:0] != 2'b11, 调用 c_ext_expand() 将 16-bit 转为 32-bit 等价指令
  2. 字段提取: opcode/rd/funct3/rs1/rs2/funct7 直接从展开后 32-bit 指令截取
  3. 立即数生成: 按 R/I/S/B/U/J 格式提取并符号扩展至 32-bit
  4. FU 分类: 根据 opcode 和 funct7 映射到 fu_type_e (ALU/MUL/DIV/MEM/FP/BRANCH/MISC)
  5. 无效指令标记: FPU 指令 (ERR-064) / Zicond (ERR-093) 设置 valid=0, 标记为无效并丢弃, 不触发非法指令异常

5.1.3 三发射仲裁逻辑

I 级采用 程序序扫描 算法, 遍历 DIB 读出的 FETCH_WIDTH 条指令, 为 ISSUE_WIDTH 个发射槽分配指令:

for slot s = 0..ISSUE_WIDTH-1:
    if stop_issue: break  // ERR-019: 前序阻塞, 后续全停
    for qi = 0..FETCH_WIDTH-1:
        if dec[qi].valid AND not_already_selected:
            fu_avail = !fu_busy[dec[qi].fu_type]
            if fu_type==FU_DIV: fu_avail &= !div_busy
            if fu_type==FU_MUL: 检查同批次无其他MUL
            batch_raw = 检查与前序已发射指令的RAW依赖
            if fu_avail AND !batch_raw:
                issue_sel[s] = qi, issue_ready[s] = 1
                issue_count++
                if fu_type==FU_BRANCH: stop_issue = 1
                break
            else:
                stop_issue = 1  // ERR-019: 阻塞后全停
                break
无效指令处理: FPU/FP-Load/FP-Store (ERR-064) 和 Zicond (ERR-093) 指令在解码阶段被标记为 valid=0。在发射仲裁中, dec[qi].valid=0 的指令被直接跳过 (不发射), 静默丢弃, 不触发非法指令异常。仅当 opcode==7'b0 && !dib_read_valid 时, I 级组装 exc_valid=1, exc_cause=EXC_ILLEGAL_INST (RTL L803-L807)。

5.5 旁路网络

旁路 mux 位于 I 级输出端, 为每个发射槽的 rs1/rs2 选择正确数据源:

// 每个发射槽 s 的旁路选择:
if bypass_rs1_hit[s]:
    d_rs1_data[s] = bypass_rs1_data[s]    // 旁路优先 (E1/E2/M/W)
else:
    d_rs1_data[s] = regfile_rs1_data[s]   // RegFile 读出值

// 特殊覆盖:
if issue_ready[s] AND !dec[issue_sel[s]].has_rs1:
    d_rs1_data[s] = 32'h0                 // LUI/AUIPC 等无 rs1 指令

// rs2 同理
旁路优先级: bypass_hit 由外部模块 (lumi_core_top) 产生, 已按 E1 > E2 > M > W 优先级编码。本模块仅做最终 2→1 mux 选择。

6. 边界条件与异常处理

6.1 边界条件清单

编号条件处理策略RTL 实现位置
BC-001 部分发射 (Partial Issue): DIB 有 6 条指令, 但仅 1~2 条可发射 (FU busy 或 RAW 阻塞) ERR-019 程序序约束: 被阻塞指令后的所有指令均停止发射。issue_count 反映实际发射数, dib_rd_ptr 仅推进 issue_count 条 L670-L728 (issue 仲裁循环)
BC-002 Stall 传播: DIB 有有效指令但 issue_count=0 (全部被阻塞), stall_out 传播到上游 fetch stall_out=1 当 dib_count>0 && issue_count==0 && !post_flush_block_r。FSM 进入 ST_STALL, DIB 读指针冻结 L732-L736 (stall 判断)
BC-003 批次内 RAW 冒险: 同一批次 (≤6 条) 中, 第 j 条的 rs1/rs2 等于第 i 条的 rd (i<j, rd≠0) tmp_batch_raw=1, 第 j 条指令不发射, 触发 stop_issue=1 停止后续。等待下一周期第 i 条结果可通过旁路获取 L696-L708 (RAW 检查内循环)
BC-004 DIB Write-Through Bypass: 同一周期 DIB 写入和读取指向相同地址 (DIB 为空或近空时) dib_bp_hit 检测写入位置与读取位置重叠, 直接转发 pd_inst_r 数据, 消除 1 周期读延迟 L218-L251 (DIB 读取组合逻辑)
BC-005 Flush 后发射抑制: flush 信号到达后, DIB 已清零, 但新 batch 尚未到达 post_flush_block_r=1 禁止所有发射, 直到 dib_count>0 (新 batch 进入 DIB) 后清零 L884-L891 (post_flush_block 寄存器)
BC-006 DIB 满保护: DIB 剩余空间不足 FETCH_WIDTH, 无法安全写入整个 batch dib_not_full=0 (保守阈值: dib_count ≤ DIB_DEPTH - FETCH_WIDTH = 26), 上游 fetch 背压 L124 (dib_not_full 赋值)
BC-007 DIB 写入偏移保守计算: 部分 slot valid, 需确保所有有效 slot 都有空间写入 dib_wr_offset 逐条检查, 若剩余空间不足以写入当前及后续所有 valid 条目, 停止累加。仅写入 i < dib_wr_offset 的条目 L156-L163 (写入偏移计算)
BC-008 分支指令后停止发射: FU_BRANCH 类型指令发射后, 后续指令不可推测执行 stop_issue=1, 当前批次后续 slot 全部为空 L717-L718 (分支停止)
BC-009 MUL 同批互斥 (SA-CM-003): 同一批次中两条 MUL 指令竞争唯一乘法器 第二条 MUL 的 tmp_fu_available=0, 触发 ERR-019 停止 L688-L693 (MUL 互斥检查)
BC-010 无效指令与非法指令处理: FPU/FP-Load/FP-Store (ERR-064), Zicond (ERR-093), 未知 opcode (opcode==0) FPU/Zicond: 解码阶段设置 dec[i].valid=0, 发射仲裁中跳过并丢弃, 不触发非法指令异常。未知 opcode (opcode==7'b0 && !dib_read_valid): I 级组装 exc_valid=1, exc_cause=EXC_ILLEGAL_INST L544-L567 (FPU valid=0), L457-L480 (Zicond valid=0), L803-L807 (exc_valid 组装)

6.3 未定义行为声明

  • OP_CUSTOM0 (Zimop): 编码为 7'b0001011 的指令当前作为 NOP 处理 (FU_MISC, 无 rs1/rs2/rd), 不触发异常 (ERR-094)。未来实现可能改变行为。
  • CBO 指令 (Zicbom/Zicbop/Zicboz): OP_FENCE 下 funct3=010/110 的 CBO/PREFETCH 作为 NOP 处理, 不触发异常。无 cache 层级时功能正确。
  • 压缩指令保留编码: c_ext_expand 对未定义的压缩编码返回 32'h0000_0000, 将被解码为非法指令。

6.4 Stall / Flush / Reset 响应

全局信号响应行为时序
stall_outDIB 读指针冻结, 不消耗指令; FSM→ST_STALL; 发射输出保持当前组合值同 cycle 生效 (组合输出)

7.1 参数约束关系

  • FETCH_WIDTH 必须 ≥ ISSUE_WIDTH (取指宽度 ≥ 发射宽度, 保证充足供给)
  • DIB_DEPTH (32) 必须 ≥ FETCH_WIDTH (保证至少一个完整 batch 可缓冲)
  • dib_not_full 阈值 = DIB_DEPTH - FETCH_WIDTH = 26, 确保上游 fetch 不被虚假背压
  • fu_busy 宽度固定 10-bit, 但仅 bit[0..7] 对应 fu_type_e 的 8 种类型

8.3 关键算法

8.3.1 DIB 写入偏移计算

dib_wr_offset = 0
for i = 0..FETCH_WIDTH-1:
    if pd_inst_valid_r[i] AND
       (dib_count + dib_wr_offset + (FETCH_WIDTH-1-i) < DIB_DEPTH):
        dib_wr_offset += 1
// 保守策略: 确保从 i 到最后一个 valid 条目都能写入, 否则截断
