4. 状态机定义

注意: 当前 FSM 为预留设计。state_reg 不驱动任何功能输出。所有功能输出 (trap_request/trap_pc/csr_we/commit_valid/regfile_wr_*/hpm_*) 由独立组合逻辑驱动。ST_IRQ 和 ST_TRAP 的下一状态均为 ST_IDLE, 行为一致。RTL 依据: lumi_writeback.sv L387-L421, state_reg 仅在 case 中用于 state_next 计算, 不驱动任何 assign 或 always_comb 输出。

4.4 死锁/活锁分析

  • 死锁防护: 所有非 IDLE 状态均有 1 cycle 无条件回退至 ST_IDLE，不存在死锁路径。ST_HALT 同样 1 cycle 回退。
  • 活锁防护: Trap/IRQ 处理期间 trap_request 拉高触发全局 flush，阻止新指令进入 W 级，避免 livelock。
  • Slot 延迟写回安全: slot2_pending 和 slot1_pending 在写入成功或无新 stall/preempt 时清除，不会无限累积。

5.3 分支结果反馈

分支指令在 W 级提交时产生 HPM 事件脉冲:

  • hpm_branch: 任意 FU_BRANCH 类型指令有效提交
  • hpm_branch_miss: 分支指令且 exc_valid 标记为误预测 (简化实现)

分支误预测的 flush 在 E1 级已完成；W 级仅做事件计数，不再触发 flush。

6.3 未定义行为声明

  • 多条 CSR 指令同周期: for 循环取最后一个有效 CSR 指令 (i=2 覆盖 i=1 覆盖 i=0)。实际调度器不应同周期发射多条 CSR 指令。
  • slot2_pending 连续累积: 若连续多周期 3 槽全满，slot2 每个周期被 stall 一次。slot2_pending 不会叠加 (每次新 stall 覆盖旧值)。
  • E2 MUL 与 DIV 同时有效: RTL 优先选择 MUL (e2_mul_valid ? e2_mul_result : e2_div_result)。实际不应出现 MUL/DIV 同周期完成。

8.3 关键算法

// 写回仲裁伪代码 (7 优先级)
function writeback_arbitrate():
    port0_busy = 0; port1_busy = 0

    // P1: Slot 0 → 端口 0
    if w_valid[0] && w_rd[0]≠0 && !w_exception[0]:
        port0.write(w_rd[0], is_csr ? csr_rdata : w_result[0])
        port0_busy = 1

    // P2: E2 MUL/DIV → 端口 1
    if e2_mul_valid || e2_div_valid:
        port1.write(e2_rd, mul_valid ? mul_result : div_result)
        port1_busy = 1
    // P3: Slot 1 → 端口 1
    else if w_valid[1] && w_rd[1]≠0 && !w_exception[1]:
        port1.write(w_rd[1], is_csr ? csr_rdata : w_result[1])
        port1_busy = 1

    // P4: Slot 2 → 端口 0 (若端口 0 空闲)
    if !port0_busy && w_valid[2] && w_rd[2]≠0 && !w_exception[2]:
        port0.write(w_rd[2], is_csr ? csr_rdata : w_result[2])
        port0_busy = 1
    // P5: Slot 2 → 端口 1 (若端口 1 空闲)
    else if !port1_busy && !e2 && w_valid[2] && w_rd[2]≠0:
        port1.write(w_rd[2], is_csr ? csr_rdata : w_result[2])
        port1_busy = 1

    // P6: slot1_pending → 端口 1 (或端口 0)
    if slot1_pending && !e2:
        if !port1_busy: port1.write(slot1_data)
        else if !port0_busy: port0.write(slot1_data)

    // P7: slot2_pending → 端口 1 (双端口均空闲)
    if slot2_pending && !port0_busy && !port1_busy && !e2:
        port1.write(slot2_data)
