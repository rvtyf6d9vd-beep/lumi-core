4.3 死锁/活锁分析

  • 死锁防护 (FPU): cycle_cnt 无条件递增, cnt_done 必然在 cycle_max 内到达
  • 死锁防护 (Vector Load/Store): 等待 vec_mem_ready, 若 Memory 永不响应则 Vector FU 永远 busy — 依赖 Memory 子系统保证响应
  • 活锁防护: FPU/Vector busy 时, Issue 级 stall 同类指令, 其他 FU 不受影响 (DEC-007-2)

6.3 Stall / Flush / Reset 响应

全局信号响应行为时序
reset_nFPU: 所有锁存/计数器/FSM 复位; Vector: 同上异步
流水线 flushFPU/Vector 当前执行不受 flush 影响 (已完成 issue 的指令必须执行完毕); 结果在 ST_DONE 后若目标指令已被 flush, 写回无效—
stallIssue 级 stall 由 busy 信号控制 (非全局 stall)同 cycle

7.1 参数约束关系

  • VLEN 必须为 2 的幂且 ≥ 128
  • FPU 延迟 (cycle_max) 硬编码在 always_comb 中, 不由参数控制
  • Vector total_steps 由 vl × SEW / VLEN 动态计算
