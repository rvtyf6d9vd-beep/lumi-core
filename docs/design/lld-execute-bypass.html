4.4 死锁/活锁分析

  • 死锁防护: DIV FSM 的 DIV_RUNNING 状态由 div_cnt 递减保证有限性 (32 cycle 必退出)。DIV_DONE 无条件回到 DIV_IDLE。ERR-057 增加 default 分支: 非法编码安全回到 DIV_IDLE。
  • 活锁防护: DIV_STALL 状态当前未使用 (直接回 IDLE), 预留扩展。DIV 执行期间 div_busy_r=1 阻止新 DIV 发射, 但其他 FU 可并行执行 (skip-stalled)。

5.3 ALU 运算流程 (E1 级, 1 cycle)

ALU 运算按 opcode 分类处理:

  1. AUIPC (opcode=0010111): result = PC + {inst[31:12], 12'h000}
  2. LUI (opcode=0110111): result = {inst[31:12], 12'h000}
  3. OPIMM (opcode=0010011): 按 funct3 分发 ADDI/SLTI/SLTIU/XORI/ORI/ANDI/SLLI/SRLI/SRAI (ERR-024)
  4. OP (opcode=0110011): 按 funct7 分发:
    • funct7=0010000: Zba SH1ADD/SH2ADD/SH3ADD
    • funct7=0100000: Zbb ANDN/ORN/XNOR
    • funct7=0000101: Zbb MIN/MAX/MINU/MAXU
    • funct7=0110000: Zbb ROL/ROR/RORI + CLZ/CTZ/CPOP
    • funct7=0110100: Zbb REV8
    • funct7=0010100: Zbb ORC.B
    • default: 标准 RV32I ALU (alu_compute 函数)
// alu_compute 函数 (组合逻辑)
function logic [31:0] alu_compute(funct3, funct7_5, op1, op2):
    shamt = op2[4:0]
    case funct3:
        000: return funct7_5 ? (op1 - op2) : (op1 + op2)   // SUB / ADD
        001: return op1 << shamt                              // SLL
        010: return {31'h0, $signed(op1) < $signed(op2)}      // SLT
        011: return {31'h0, op1 < op2}                        // SLTU
        100: return op1 ^ op2                                  // XOR
        101: return funct7_5 ? $unsigned($signed(op1) >>> shamt) : (op1 >> shamt)  // SRA/SRL
        110: return op1 | op2                                  // OR
        111: return op1 & op2                                  // AND

5.4 MUL 执行流程 (E2 级, 1-cycle 延迟)

MUL 在 E1 级进行组合逻辑计算 (mul_full 64-bit), E2 级寄存器锁存输出:

funct3操作乘法类型E2 输出

001 (MULH)高 32 位有符号 × 有符号mul_full[63:32]
010 (MULHSU)高 32 位有符号 × 无符号mul_full[63:32]

5.5 DIV 执行流程 (FSM, 恢复除法)

DIV 采用恢复除法 (restoring division), 每 cycle 处理 1 bit, 共 32 cycle + init + done:

// DIV_RUNNING 单步: 恢复除法
sub_result = {remainder[30:0], dividend[cnt-1]} - {1'b0, divisor}
if (!sub_result[32]):  // 够减
    remainder = sub_result[31:0]
    quotient  = {quotient[30:0], 1'b1}
else:                  // 不够减
    remainder = {remainder[30:0], dividend[cnt-1]}
    quotient  = {quotient[30:0], 1'b0}
cnt = cnt - 1

// DIV_DONE: 符号修正
if (div_is_rem):
    result = (div_signed && div_remain_negate) ? -remainder : remainder
else:
    result = (div_signed && div_quot_negate) ? -quotient : quotient

// 除零覆盖 (SA-CM-010):
if (divisor == 0):
    result = div_is_rem ? dividend : 0xFFFFFFFF  // RISC-V 规范

6.3 未定义行为声明

  • DIV_STALL 状态: 当前未使用, 进入后直接回 DIV_IDLE。若未来扩展 stall 逻辑, 需确保有退出路径。
  • 3 条 MEM 指令同时发射: 仅 2 个 LSU 端口, 第 3 条 MEM 不会被分配到任何端口 (地址/数据保持 0)。I 级应避免此类发射。
  • FPU 旁路命名空间: fpu_rd 与整数 rd 共享 5-bit 编码空间。当 FPU 使用独立 FRF 时, 需外部逻辑区分, 否则可能产生虚假旁路。
  • CLZ/CTZ/CPOP 选择: R-type 使用 rs2 field 区分 (00000=CLZ, 00001=CTZ, 00010=CPOP), 其他 rs2 值在 funct3=001 路径下: R-type default 分支为 ROL (rs2[0]=1), I-type default 分支为 alu_compute。
