4. 状态机定义

PLIC 模块无显式 FSM。中断处理流程通过组合逻辑仲裁 + 寄存器状态管理实现， Claim/Complete 协议通过 claimed_valid_r 标志位控制。

4.1 Claim/Complete 状态模型

状态名条件含义进入条件
IDLEclaimed_valid_r[h] == 0无 pending claim，可接受新中断reset / Complete 写入
CLAIMEDclaimed_valid_r[h] == 1已 claim 中断，等待 Completeirq_claim 有效且 irq_above_threshold

4.2 状态转移图

stateDiagram-v2
    [*] --> IDLE : reset
    IDLE --> CLAIMED : irq_claim[h] && irq_above_threshold[h] && !claimed_valid_r[h]
    CLAIMED --> IDLE : MMIO write complete (plic_addr=0x104+h*8)
    CLAIMED --> CLAIMED : 保持 (无 complete 写入)
    IDLE --> IDLE : 无 claim 请求

4.3 状态转移条件表

源状态目标状态触发条件输出动作
IDLECLAIMEDirq_claim[h] && irq_above_threshold[h] && !claimed_valid_r[h]claimed_src_r[h] ≤ best_id[h]; pending_r[best_id] ≤ 0
CLAIMEDIDLEplic_we && addr==0x104+h*8claimed_valid_r[h] ≤ 0

4.4 死锁/活锁分析

  • 死锁防护: Claim 后 pending 位清零，同源新中断可在 Complete 后重新 pending。若软件不执行 Complete，同源中断被阻塞但其他源不受影响 (per-src pending)。
  • 活锁防护: 仲裁为纯组合逻辑，无迭代收敛问题。Claim 条件要求 !claimed_valid_r[h]，防止重复 claim。

5. 数据通路与时序

5.1 关键数据通路

graph LR
    A[ext_irq_in 32-bit] --> B[pending_r 更新]
    B --> C[per-hart 仲裁器]
    D[enable_r per-hart] --> C
    E[priority_r 3-bit x 32] --> C
    C --> F[best_id / best_priority]
    F --> G{priority > threshold?}
    G -->|Yes| H[meip=1, irq_id=best_id]
    G -->|No| I[meip=0]
    H --> J[irq_claim → Claim 锁存]
    J --> K[MMIO Complete → 清除 claim]

5.1.1 优先级仲裁逻辑

仲裁采用 双层嵌套组合逻辑：

  1. 外层循环: per-hart (h = 0..NUM_HART-1)
  2. 内层循环: 遍历所有中断源 (i = 1..NUM_SRC-1, 跳过源 0)
  3. 比较条件: pending_r[i] && enable_r[h][i] 作为使能
  4. 优先级规则: 高优先级优先；同优先级高 ID 优先
  5. Threshold 过滤: best_priority > threshold_r[h] 时才输出 MEIP

5.2 流水线级间时序

Cycle输入内部状态输出备注

7. 可配置参数

参数名类型默认值取值范围影响描述
NUM_SRCint3232~960 (2 的幂次)外部中断源数量；影响 pending/enable/priority 数组大小和仲裁循环
NUM_HARTintlumi_pkg::NUM_HARTS (1)1~8Hart 数量；影响 per-hart enable/threshold/claim 数组和 MEIP 输出位宽

7.1 参数约束关系

  • NUM_SRC 必须为 32 的倍数，且 ≥ 32
  • NUM_HART 决定 HART_IDX_W = (NUM_HART==1) ? 1 : $clog2(NUM_HART)
  • ENABLE_BASE = 0x084，THRESHOLD_BASE = 0x100，地址空间需满足 ENABLE_BASE + NUM_HART*4 ≤ THRESHOLD_BASE
  • irq_id 位宽为 5-bit per hart (packed)，最大支持 32 个源
  • 优先级位宽固定为 3-bit (0~7 级)

8.3 关键算法

// PLIC 优先级仲裁伪代码 (per-hart, 组合逻辑)
for h = 0..NUM_HART-1:
    best_id = 0; best_priority = 0; has_irq = 0

    for i = 1..NUM_SRC-1:  // 跳过源 0
        if pending_r[i] AND enable_r[h][i]:
            if priority_r[i] > best_priority:
                best_priority = priority_r[i]
                best_id = i
                has_irq = 1
            elif priority_r[i] == best_priority AND i > best_id:
                best_id = i  // 同优先级高 ID 优先
                has_irq = 1

    // Threshold 过滤
    irq_above_threshold[h] = has_irq AND (best_priority > threshold_r[h])
    meip[h] = irq_above_threshold[h]
    irq_id[h*5+:5] = irq_above_threshold[h] ? best_id : 0
