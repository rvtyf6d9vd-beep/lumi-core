1. 概述与范围

1.1 模块职责

Register File 模块负责 Lumi-Core 的三类寄存器堆的存储阵列、端口管理和访问控制：

  • 整数寄存器堆 (IRF)：32 entry × 32 bit, 3R/2W 端口
  • 浮点寄存器堆 (FRF)：32 entry × 64 bit, 2R/1W 端口
  • 向量寄存器堆 (VRF)：32 entry × VLEN bit, 多 Bank 结构, 每 Bank 1R/1W
GEM5 映射: GEM5 MinorCPU 隐式建模寄存器堆 (通过 ISA 描述寄存器数量)。本设计文档定义 RTL 实现的物理寄存器堆规格。

2. 功能描述

2.1 整数寄存器堆 (IRF)

2.1.1 规格

参数规格描述
条目数32 entry (x0~x31)RISC-V RV32I 标准
位宽32 bitXLEN=32
读端口3R支持三发射同时读 3 个源操作数
写端口2W支持同时写回 2 个结果
x0 处理读恒为 0, 写忽略节省 1 个写端口使能和存储单元
读延迟1 cycle组合逻辑读 (SRAM 或触发器阵列)
写延迟1 cycle同步写入

2.1.2 端口压力分析 D-009

D-009 三发射端口压力:
读端口需求 (最坏情况): 3 条指令 × 2 源寄存器 = 6 个读请求，但仅 3R 端口可用。实际分析：
• 场景 A (典型)：1 条 Load (rs1) + 1 条 ALU (rs1, rs2) + 1 条 Store (rs1, rs2) = 5 读请求 → 需 2 cycle
• 场景 B (最坏)：3 条 ALU 各需 rs1+rs2 且全不重叠 = 6 读请求 → 需 2 cycle
• 场景 C (常见)：指令间共享 rs1/rs2 (如循环计数器) = 去重后 3~4 读请求 → 可 1 cycle

写端口需求: 最多 2 条指令同时到达 W 级写回，2W 端口满足。极端 3 条同时到达时 stall 1 cycle。

仲裁策略: 程序顺序优先 (Slot 0 > Slot 1 > Slot 2)，端口不足时后续指令 stall 1 cycle。

2.1.3 实现方案选择

方案优势劣势选择
触发器阵列 (Flip-Flop)速度快, 多端口简单, 1 cycle 读面积大 (~0.01mm²)基准方案 (32 entry 面积可控)
SRAM (6T)面积小多端口需 bank 复制, 读延迟可能 >1 cycle备选 (面积受限时)
寄存器文件 IP (Foundry)PPA 最优依赖 IP 可用性, 端口数受限量产方案
决策: 32 entry × 32 bit 的 IRF 面积较小 (~1024 bit)，采用触发器阵列方案在 TSMC 28HPC 下面积约 0.008mm²，可满足 1GHz 时序要求。

2.1.4 x0 零寄存器优化

// x0 硬连线逻辑
function irf_read(addr, port):
    if addr == 0:
        return 0x00000000  // x0 读恒为 0
    else:
        return reg_array[addr]

function irf_write(addr, data, we):
    if addr == 0:
        // x0 写忽略 (节省写端口使能)
        return
    elif we:
        reg_array[addr] = data

2.2.1 规格

参数规格描述

2.2.2 NaN-Boxing 逻辑

// NaN-Boxing: FP32 值写入 64-bit 寄存器
function frf_write_fp32(addr, data32):
    reg_array[addr] = {0xFFFFFFFF, data32}  // 高位填 1

// NaN-Boxing 检测: 读取时检查高位
function frf_read_fp32(addr):
    val64 = reg_array[addr]
    if val64[63:32] == 0xFFFFFFFF:
        return val64[31:0]  // 合法 NaN-boxed FP32
    else:
        return canonical_NaN()  // 非法 NaN-boxing, 返回 canonical NaN

// FP64 直接读写 (无 NaN-Boxing)
function frf_write_fp64(addr, data64):
    reg_array[addr] = data64

function frf_read_fp64(addr):
    return reg_array[addr]

// VRF Bank 分配伪代码
function vrf_read(vreg, element_idx):
    bank_idx = vreg % NUM_BANKS
    if bank_conflict(bank_idx):
        stall(1)  // Bank 冲突, stall 1 cycle
        return vrf_read(vreg, element_idx)  // 重试
    else:
        return bank[bank_idx].read(vreg, element_idx)

4. 数据通路

4.1 寄存器堆读写时序

Cycle N:
  D 级: irf_rs1_addr[0..2] 送出 → 寄存器堆读
        irf_rs2_addr[0..2] (分时或端口仲裁)
  D 级末: irf_rs1_data[0..2] / irf_rs2_data[0..2] 返回
        (与旁路网络选择后输出到 I 级)

Cycle N+1:
  E1 级: 使用 rs1/rs2 数据运算
  W 级: irf_wr_addr[0..1] + irf_wr_data[0..1] 写入
        (写后下一 cycle 可读, 或通过旁路立即转发)

4.2 读写冲突处理

当同一 cycle 内读地址与写地址相同时 (RAR/WAR 冒险)：

场景处理策略描述
读优先 (Read-First)返回旧值适用于顺序执行, 写在同一 cycle 末尾
写优先 (Write-First)返回新值减少旁路需求, 但时序更紧
旁路转发 (Forwarding)W 级写数据同时转发到 D 级读输出Lumi-Core 采用此方案
// 读写冲突旁路转发
function irf_read_with_forward(addr):
    // 检查同 cycle 是否有写操作到相同地址
    for wr_port in [0, 1]:
        if irf_wr_en[wr_port] and irf_wr_addr[wr_port] == addr:
            return irf_wr_data[wr_port]  // 写转发
    return reg_array[addr]  // 无冲突, 正常读
