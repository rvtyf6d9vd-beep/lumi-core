4.5 死锁/活锁分析

  • 死锁防护（AXI FSM）: 所有传输状态均有明确的完成退出条件（arready/awready/wready/bvalid/rvalid+rlast）。AXI 协议保证 Slave 最终响应。若 Slave 挂死，FSM 将永久停留在等待状态（设计上依赖 SoC 超时机制）。
  • 活锁防护（仲裁）: 固定优先级仲裁可能导致低优先级 Master（Trace）饥饿。当前设计接受该行为，因 Trace 为异步写出、FIFO 缓冲可吸收突发。
  • 死锁防护（CDC FIFO）: FIFO DEPTH=4，满时阻止写入（wr_en && !wr_full），空时阻止读出。格雷码指针保证跨域单 bit 跳变，避免亚稳态导致的虚假空/满。

6. 边界条件与异常处理

6.1 边界条件清单

编号条件处理策略验证覆盖
BC-001PMA 地址落在未映射区域（如 0x0003_0000~0x7FFF_FFFF）pma_result = PMA_FAULT，pmp_check_req=0（短路跳过 PMP），route_*=0地址扫描 0x0003_0000, 0x4000_0000, 0x7FFF_FFFF, 0xF001_0000
BC-002PMA MMIO 区域 FETCH 访问（NX 保护）nx_fault=1 → PMA_FAULT（对应 mcause=1 Instruction Access Fault）pma_addr=0xC000_0000, access_type=FETCH
BC-003PMA TCM/MMIO 区域 AMO 访问amo_fault=1 → PMA_FAULT（amo_type=2'b00 区域不支持 AMO）pma_addr=0x0000_0000/0xC000_0000, access_type=AMO
BC-004PMA 地址非对齐（如半字访问奇数地址）align_fault=1 → PMA_MISALIGNED（对应 mcause=0/4/6）pma_addr[0]=1 + access_type=Halfword
BC-0054 路 Master 同时请求（DC + IC + NC + Trace）固定优先级：DC 获得仲裁，其余等待。DC evict 优先于 DC refill。全组合请求测试
BC-006CDC FIFO 满时 FSM 尝试发送axi_awready_core = !aw_full：FSM 停留在当前 *_AR/*_AW 状态，等待 FIFO 空间。背压场景：bus 侧长时间不 ready
BC-007DC evict 多 beat 写（4 beat × 16B）evict_beat_cnt 计数 0→3，仅在 beat_cnt == len_reg 时 wlast=1。进入 ST_DC_EVICT_W 时清零计数。完整 4 beat burst 验证
BC-008NC 写数据扩展（32bit → 128bit）wdata_reg = {4{nc_access_wdata}} 复制到 128bit，但 wstrb={4'hF,12'h0} 仅低 4 byte 有效。NC write + 各字节对齐
BC-009PMA 区域边界地址（TCM_END=0x0003_0000, DDR_END=0xC000_0000, MMIO_END=0xF001_0000）比较为 < END（exclusive），0x0003_0000 不属于 TCM，0xC000_0000 属于 MMIO 不属于 DDR。边界值 ±1 扫描
BC-010复位期间所有信号FSM→ST_IDLE, ack→0, addr_reg→0, CDC FIFO 指针→0（全 0），格雷码 0→0（空状态正确）复位后立即发起请求

6.3 未定义行为声明

  • pma_access_type 未使用编码（3'b100~3'b111）: RTL 中 pmp_check_type default 分支返回 read(3'b010)。PMA 检查不会匹配 FETCH/LOAD/STORE/AMO，故不触发 nx_fault 或 amo_fault，结果为 PMA_PASS（若地址命中合法区域）。属于未定义行为，验证应 cover。
  • ST_FFP_ACCESS 状态: 当前 RTL 中无进入路径（ST_IDLE 不会跳转到该状态），state_next = ST_IDLE 直通。若因 SEU 进入，1 cycle 后恢复。
  • DC evict + DC refill 同时请求: RTL 中 arb_dc_grant = dc_refill_req_sync || dc_evict_req_sync，dc_evict_req_sync 优先检查，故 evict 优先。

7.2 参数约束关系

  • AXI_ID_W ≥ 2：需至少 2 bit 以编码 4 个 Master ID（0~3）
  • DEPTH 必须为 2 的幂次：格雷码指针要求 PTR_WIDTH = $clog2(DEPTH)
  • STAGES ≥ 2：CDC 2-FF 同步器最低 2 级，低于此值无法保证亚稳态消除
  • AXI_DATA_W 必须为 8 的倍数：wstrb 位宽 = AXI_DATA_W / 8
