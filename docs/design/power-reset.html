1. 概述与范围

1.1 模块职责

低功耗 & 复位 & 时钟域模块管理 Lumi-Core 的功耗状态、时钟门控和复位域：

  • 低功耗管理：WFI/WRS 指令触发的低功耗状态 (Active/Sleep/Deep Sleep)
  • Clock Gating：细粒度 ICG 门控 + WFI 触发门控 + SoC 强制门控
  • 复位域：Cold/Warm/Debug/Power-on 四级复位域划分
  • 时钟域：clk_core / clk_bus / clk_tcm / tck / clk_trace 多域管理
  • UPF 电源约束：电源域划分 + 隔离策略 + Retention 寄存器
  • 启动序列：复位释放 → Boot 初始化流程

2.1.2 WFI 执行流程 审查约束: WFI 延迟

// WFI 执行状态机
//
// ┌─────────┐  WFI decode  ┌──────────┐  pipeline  ┌─────────┐
// │ Active  │─────────────►│ WFI_ARM  │───────────►│  Sleep  │
// └─────────┘              └──────────┘  drain     └────┬────┘
//      ▲                                                │
//      │         wake_event                             │
//      └────────────────────────────────────────────────┘

function wfi_execute():
    // 1. WFI 在 D 级解码
    // 2. WFI 提交 (W 级) 后进入低功耗:
    //    a. 等待流水线排空 (F2/D/I/E1/E2/M 全部完成)
    //    b. 关闭 Core Clock 域 (clk_gate_enable = 1)
    //    c. 仅保留中断检测逻辑运行
    
    if pending_interrupt():
        // WFI 作为 NOP: 有待处理中断时不进入 Sleep
        return
    
    pipeline_drain()  // 等待所有 in-flight 指令完成
    core_clk_gate(1)  // 关闭 Core Clock
    power_state = SLEEP

// WFI 唤醒流程
function wfi_wake(wake_source):
    // 唤醒源: 中断 / NMI / Debug halt / SoC 强制唤醒
    core_clk_gate(0)  // 恢复 Core Clock
    
    // 唤醒序列:
    // cycle 1: clk_gate 释放, 时钟恢复
    // cycle 2: F1 级开始取指 (从 WFI 下一条指令 PC)
    // cycle 3: F2 级 ICache 读出
    // cycle 4~8: D/I/E1/E2/M/W 流水线填充
    // 首条指令在 WFI 唤醒后 ≤8 cycle 到达 W 级
    
    power_state = ACTIVE
    // 如果是中断唤醒, 进入中断处理流程
    if wake_source == INTERRUPT:
        handle_interrupt()
审查约束 — WFI 延迟:
  • 要求: WFI 唤醒后首条指令到执行 ≤8 cycle (platform-spec §8.9.2)
  • 分解: 1 cycle (clk gate release) + 1 cycle (F1) + 1 cycle (F2) + 1 cycle (D) + 1 cycle (I) + 1 cycle (E1) + 1 cycle (E2) + 1 cycle (M→W) = 8 cycle
  • ICache 影响: 如果唤醒后首条指令 ICache miss，额外增加 Cache refill 延迟 (不在 8 cycle 预算内)
  • TCM 无影响: TCM 访问延迟恒定 (0/1 cycle)，WFI 不影响 TCM 内容
  • WRS 差异: WRS 等待 reservation set，唤醒条件为内存写入到监控地址，唤醒延迟相同
  • 验证: Cover Group cg_wfi_latency 覆盖唤醒 cycle 计数

2.1.3 WRS (Wait for Reservation Set)

// WRS 执行流程 (Zawrs 扩展)
function wrs_execute():
    // 1. WRS 检查当前 hart 的 reservation 是否有效
    // 2. 如果 reservation 有效 → 进入等待 (类似 WFI)
    // 3. 等待条件: 其他 hart/设备写入到 reservation 地址
    // 4. 唤醒: reservation 被清除 (外部写入) 或中断
    
    if not reservation_valid:
        return  // reservation 无效, WRS 作为 NOP
    
    pipeline_drain()
    core_clk_gate(1)  // 进入 Sleep
    // 唤醒条件: reservation_invalidate 信号

2.2.2 细粒度门控逻辑

// ICG (Integrated Clock Gating) 控制
//
// 每级流水线寄存器前有 ICG cell:
//   clk_gated = clk_core & icg_enable
//
// icg_enable 生成:
//   icg_enable_F1 = not (stall_F1 or flush_F1 or sleep_mode)
//   icg_enable_F2 = not (stall_F2 or flush_F2 or sleep_mode)
//   icg_enable_D  = not (stall_D or sleep_mode)
//   ...
//   icg_enable_W  = not (sleep_mode)  // W 级在 sleep 前完成所有指令
//
// 模块级门控:
//   icg_enable_FPU = not (fpu_idle and fpu_no_pending)
//   icg_enable_VEC = not (vec_idle and vec_no_pending)
//   icg_enable_TCM = not (tcm_no_access and tcm_no_dma)

2.3.2 跨时钟域同步

// CDC (Clock Domain Crossing) 策略
//
// 1. clk_core ↔ clk_bus (同步/分频关系):
//    - 使用 2-FF 同步器传递控制信号
//    - 使用 Async FIFO 传递数据 (深度可配, 默认 4 entry)
//    - AXI handshake 信号使用 2-FF 同步
//
// 2. tck ↔ clk_core (完全异步):
//    - DMI 接口使用 req/ack 异步握手协议
//    - dmi_req → 2-FF 同步到 clk_core → 处理 → 2-FF 同步 dmi_ack 回 tck
//    - 延迟: 2~4 clk_core cycle + 2~4 tck cycle
//
// 3. clk_core ↔ clk_trace (可选, 通常同步):
//    - 如果 clk_trace = clk_core: 直接连接
//    - 如果异步: 2-FF 同步 + Async FIFO (默认 8 entry)
//
// CDC 验证: LUMI-QUAL-001 覆盖 (SpyGlass CDC / Conformal CDC)

// 复位释放顺序 (确保系统稳定启动)
//
// 1. por_n 释放 (Power-on Reset, 最先)
// 2. reset_n 释放 (Cold Reset)
// 3. bus_rst_n 先释放 (AXI 互联稳定)
// 4. core_rst_n 后释放 (Hart 开始取指, 此时 AXI 已就绪)
// 5. ndmreset 可在任意时刻由 Debug Module 触发
//
// 复位释放同步器:
//   每个复位域使用 2-FF 释放同步器, 防止复位释放时的亚稳态
//   reset_sync = FF2(reset_n_async, clk_target_domain)

2.5 启动序列 LUMI-BOOT-002 LUMI-BOOT-003

// 启动序列 (Cold Reset 后)
//
// 1. reset_n 拉低 → 全模块复位
// 2. reset_n 拉高 → Hart 从复位向量开始执行:
//    - PC = reset_vector (可配: 0x0000_0000 ITCM / 0x8000_0000 DDR / Boot ROM)
//    - mstatus.MPP = M-mode, mstatus.MIE = 0
//    - mtvec = 0 (待软件初始化)
//    - PMP: 全 Region OFF (待软件配置)
//
// 3. M-mode 初始化 (Bootloader/固件):
//    a. 配置 PMP (Smepmp: mseccfg.MML=1, MMWP=1)
//    b. 配置 mtvec (vectored 模式, 中断向量表)
//    c. 初始化 TCM: ITCM/DTCM 加载 Bootloader
//    d. 使能 Cache: I-Cache/D-Cache 开启
//    e. 初始化 CLIC/PLIC
//
// 4. 可选: 切换到 S-mode (RTOS) 或 U-mode (应用)
//
// Boot ROM (可选, LUMI-BOOT-003):
// - 内置 4~32KB Boot ROM, 映射到复位向量地址
// - 包含最小化初始化代码 (PMP 配置 + TCM 加载 + 跳转)
// - 支持固件签名验证 (RSA-2048/ECDSA-P256, LUMI-SEC-001)

// 隔离策略 (跨电源域信号)
// - 跨域信号使用 isolation cell (默认 clamp 0)
// - iso_en 信号由 Always-On 域控制
// - Deep Sleep 进入时: iso_en = 1 (clamp 所有跨域信号)
// - Deep Sleep 退出时: iso_en = 0 (恢复正常信号)

// Retention 寄存器 (Deep Sleep 状态保持)
// 关键 CSR 使用 retention register:
//   - mstatus, mepc, mtvec, mcause
//   - pmpcfg0~3, pmpaddr0~7

// Retention 寄存器在 Deep Sleep 时由 VDD_AON 供电保持

4. 数据通路

4.1 WFI 低功耗状态机

// WFI 状态机 (RTL 框架)
//
// States:
//   ACTIVE    — 正常运行
//   DRAIN     — 流水线排空中
//   SLEEP     — Core Clock 门控, 等待唤醒
//   WAKEUP    — 时钟恢复, 流水线重新填充
//
// Transitions:
//   ACTIVE → DRAIN:   wfi_commit (WFI 指令在 W 级提交)
//   DRAIN → SLEEP:    pipeline_empty (所有 in-flight 指令完成)
//   SLEEP → WAKEUP:   wake_event (中断/NMI/Debug/SoC)
//   WAKEUP → ACTIVE:  wakeup_done (首条指令进入 F1)
//   ACTIVE → ACTIVE:  wfi_commit + pending_interrupt (WFI 作为 NOP)
//
// 输出:
//   core_clk_gate_en = (state == SLEEP)
//   wfi_active       = (state == SLEEP or state == DRAIN)

4.2 复位释放同步器

// 复位释放同步器 (per-domain)
//
// module reset_sync (
//     input  wire clk,        // 目标时钟域
//     input  wire rst_async,  // 异步复位 (低有效)
//     output wire rst_sync    // 同步复位 (低有效)
// );
//     reg ff1, ff2;
//     always @(posedge clk or negedge rst_async) begin
//         if (!rst_async) begin
//             ff1 <= 1'b0;
//             ff2 <= 1'b0;
//         end else begin
//             ff1 <= 1'b1;
//             ff2 <= ff1;
//         end
//     end
//     assign rst_sync = ff2;
// endmodule
//
// 使用:
//   reset_sync u_core_rst (.clk(clk_core), .rst_async(reset_n), .rst_sync(core_rst_n));
//   reset_sync u_bus_rst  (.clk(clk_bus),  .rst_async(reset_n), .rst_sync(bus_rst_n));
//   // bus_rst_n 先释放 (通过延迟链), core_rst_n 后释放

6.2 关键验证场景

  • WFI 延迟: 精确测量 WFI 唤醒到首条指令提交 cycle 数 ≤ 8
  • WFI 竞争: WFI 执行同时中断到达 (WFI 作为 NOP vs 进入 Sleep)
  • 复位顺序: 验证 bus_rst_n 先于 core_rst_n 释放
  • Debug 复位: ndmreset 在 Sleep 状态触发 (应唤醒并复位 Hart)
  • Deep Sleep: 完整进入/退出序列 + Retention 验证
  • CDC: 全跨域路径 SpyGlass CDC 验证通过
