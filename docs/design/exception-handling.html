1. 概述与范围

1.1 模块职责

异常处理模块管理 Lumi-Core 的全部同步异常和NMI，实现精确异常模型：

  • 异常检测：在流水线各级检测同步异常 (地址未对齐、访问违例、非法指令等)
  • 精确异常：异常指令之前的所有指令已完成提交，异常指令及之后的指令全部丢弃
  • 陷阱委托 (Trap Delegation)：M-mode 可将异常/中断委托到 S-mode 处理
  • NMI (Smrnmi)：不可屏蔽中断的可恢复处理机制
  • 异常优先级仲裁：同一周期多个异常事件的优先级排序

2.2 异常优先级仲裁 审查约束: 异常优先级

2.2.1 优先级规则

同一周期可能存在多个待处理异常/中断，按以下优先级处理：

优先级类型描述
1 (最高)Instruction Address Breakpoint取指断点 (在取指前检测)
2Instruction Page Fault / Access Fault / Misaligned取指异常 (F1/F2 级)
3Illegal Instruction / Environment Call / Breakpoint解码级异常 (D 级)
4Load/Store Address Misaligned地址计算级异常 (E1 级)
5Load/Store Access Fault内存访问级异常 (M 级)
6Reset复位不可抢占
7NMI (Smrnmi)高优先级中断
8外部中断 (PLIC → MEIP)CLIC 仲裁后的外部中断
9CLIC 本地中断本地中断源
10软件中断 (MSI)软件触发
11 (最低)定时器中断MTIP
审查约束 — 异常优先级:
  • 规则 1: 同步异常 (与当前指令关联) 始终优先于异步中断
  • 规则 2: 同步异常之间按流水线级顺序 (早期级优先)：取指 > 解码 > 执行 > 内存
  • 规则 3: 同一指令同一级的多个异常 (如地址未对齐 + 访问违例) 按 mcause 编号排序 (小编号优先)
  • 规则 4: NMI 优先于所有普通中断，但低于同步异常
  • 实现: 优先级编码为组合逻辑，W 级汇总所有 pending 异常后选取最高优先级
  • 验证: Cover Group cg_exception_priority 覆盖多异常同时到达场景

2.2.2 优先级仲裁伪代码

// W 级异常优先级仲裁
function exception_arbitrate():
    // 收集各级异常
    fetch_exception   = get_fetch_exception()      // F1/F2: mcause 0/1
    decode_exception  = get_decode_exception()     // D: mcause 2/3/8/11
    exec_exception    = get_exec_exception()       // E1: mcause 4/6
    mem_exception     = get_mem_exception()        // M: mcause 5/7
    nmi_pending       = get_nmi_status()           // Smrnmi
    irq_pending       = get_highest_irq()          // 来自 clic-plic.html

    // 优先级: 同步 > NMI > 中断
    if fetch_exception.valid:
        return {TRAP, fetch_exception.cause, fetch_exception.tval}
    elif decode_exception.valid:
        return {TRAP, decode_exception.cause, decode_exception.tval}
    elif exec_exception.valid:
        return {TRAP, exec_exception.cause, exec_exception.tval}
    elif mem_exception.valid:
        return {TRAP, mem_exception.cause, mem_exception.tval}
    elif nmi_pending:
        return {NMI, nmi_cause, 0}
    elif irq_pending.valid:
        return {IRQ, irq_pending.id, 0}
    else:
        return {NONE, 0, 0}

2.3 精确异常模型

Lumi-Core 实现精确异常 (Precise Exception)：异常指令之前的所有指令已完成提交，异常指令及之后的指令全部丢弃。

// 精确异常保证
//
// 指令流:  ... A(committed) | B(exception) | C(not fetched) ...
//                         ▲              ▲
//                    已提交 (状态已更新)  异常指令 (未提交, 无副作用)
//
// 8 级顺序流水线天然支持精确异常:
// - 顺序提交 (W 级按程序顺序)
// - 异常指令在 W 级检测, 此时前指令已全部提交
// - 异常指令本身在 W 级被拦截, 不执行写回
// - F2/D/I/E1/E2/M 级中后续指令全部 flush

2.4 陷阱委托 (Trap Delegation)

2.4.1 委托 CSR

CSR地址描述
medeleg0x302M-mode 异常委托位图 (bit = mcause 异常码)
mideleg0x303M-mode 中断委托位图 (bit = 中断码)

2.4.2 委托流程

// 陷阱委托判断
function trap_delegate(cause, is_interrupt):
    if is_interrupt:
        delegated = mideleg[cause]  // 中断委托
    else:
        delegated = medeleg[cause]  // 异常委托

    if delegated and current_privilege <= S:
        // 委托到 S-mode
        scause = {is_interrupt, cause}
        sepc   = faulting_pc
        stval  = trap_value
        sstatus.SPIE = sstatus.SIE
        sstatus.SPP  = current_privilege
        sstatus.SIE  = 0
        PC = stvec  // S-mode trap handler
    else:
        // 在 M-mode 处理
        mcause = {is_interrupt, cause}
        mepc   = faulting_pc
        mtval  = trap_value
        mstatus.MPIE = mstatus.MIE
        mstatus.MPP  = current_privilege
        mstatus.MIE  = 0
        PC = mtvec  // M-mode trap handler

2.5 NMI 处理 (Smrnmi)

2.5.1 NMI 进入

// NMI 进入流程
function nmi_enter():
    mnepc    = current_pc       // 保存被中断 PC
    mncause  = nmi_cause_code   // NMI 原因码
    mnstatus.NMPP = mstatus.MPP // 保存特权级
    mnstatus.NMP  = mstatus.MIE // 保存 MIE
    mstatus.MPP  = M_MODE       // 强制切到 M-mode
    mstatus.MIE  = 0            // 禁止所有中断
    PC = mtvec.NMI_VECTOR       // NMI 入口地址

2.5.2 NMI 返回 (mnret)

// mnret 指令执行
function mnret():
    PC = mnepc                    // 恢复被中断 PC
    mstatus.MPP = mnstatus.NMPP  // 恢复特权级
    mstatus.MIE = mnstatus.NMP   // 恢复 MIE
    // NMI 与普通中断互斥: NMI 处理期间不接受新 NMI

2.6 MRET / SRET 返回逻辑

// MRET: Machine-mode trap return
function mret():
    PC = mepc
    mstatus.MIE = mstatus.MPIE   // 恢复中断使能
    mstatus.MPP = U_MODE if mstatus.MPP != M_MODE else M_MODE
    mstatus.MPIE = 1

// SRET: Supervisor-mode trap return
function sret():
    PC = sepc
    sstatus.SIE = sstatus.SPIE
    sstatus.SPP = U_MODE
    sstatus.SPIE = 1

4. 数据通路与控制

4.1 Trap 数据通路

// Trap 完整数据通路
//
// 流水线各级异常信号
//     │
//     ▼
// [W 级: 异常汇总 + 优先级仲裁]
//     │
//     ▼
// [委托判断: medeleg/mideleg]
//     │
//     ├──→ M-mode trap: mepc/mcause/mtval/mstatus → PC=mtvec
//     │
//     ├──→ S-mode trap: sepc/scause/stval/sstatus → PC=stvec
//     │
//     └──→ NMI: mnepc/mncause/mnstatus → PC=NMI_VECTOR
//
// [Pipeline Flush]: F2/D/I/E1/E2/M 级全部 flush
// [PC Mux]: 选择 trap PC (mtvec/stvec/NMI/MRET/SRET/mnret)
