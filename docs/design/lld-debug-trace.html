4. 状态机定义

4.1 Debug 主 FSM — dbg_state_e (3-bit, 6 states)

状态名编码含义进入条件
ST_RUN3'b000正常运行reset / resume
ST_HALT_REQ3'b001等待 W 级精确 halthaltreq && !debug_locked
ST_HALTED3'b010已 halt, 等待命令commit_valid || halted_ack
ST_RESUME_REQ3'b011恢复执行resumereq
ST_SINGLE_STEP3'b100单步执行step_mode && !debug_locked
ST_TRIGGER3'b101Trigger 命中 halttrigger_hit && !debug_locked

4.2 Debug FSM 状态转移图

stateDiagram-v2
    [*] --> ST_RUN : reset
    ST_RUN --> ST_HALT_REQ : haltreq && !locked
    ST_RUN --> ST_TRIGGER : trigger_hit && !locked
    ST_HALT_REQ --> ST_HALTED : commit_valid || halted_ack
    ST_HALTED --> ST_RESUME_REQ : resumereq
    ST_HALTED --> ST_SINGLE_STEP : step_mode && !locked
    ST_RESUME_REQ --> ST_RUN : (always)
    ST_SINGLE_STEP --> ST_HALTED : single_step_done
    ST_TRIGGER --> ST_RESUME_REQ : resumereq

4.3 DMUI 握手 FSM — dmui_state_e (2-bit, 3 states)

状态名编码含义进入条件
DMUI_IDLE2'b00等待 DMI 请求reset / valid 撤除
DMUI_PROC2'b01处理 DMI 读写dmi_req_posedge_core
DMUI_DONE2'b10应答完成PROC 完成 (1 cycle)

4.4 Trace 主 FSM — trace_state_e (2-bit, 4 states)

状态名编码含义进入条件
ST_IDLE2'b00Trace 空闲reset / trace_enable=0
ST_CAPTURE2'b01捕获 commit 信息trace_enable=1
ST_ENCODE2'b10Nexus 编码 + FIFO 写入commit_valid
ST_FLUSH2'b11AXI burst 写出FIFO 有数据
stateDiagram-v2
    [*] --> ST_IDLE : reset
    ST_IDLE --> ST_CAPTURE : trace_enable
    ST_CAPTURE --> ST_ENCODE : commit_valid
    ST_CAPTURE --> ST_IDLE : !trace_enable
    ST_ENCODE --> ST_FLUSH : !fifo_full || burst_full
    ST_ENCODE --> ST_CAPTURE : fifo_full
    ST_FLUSH --> ST_CAPTURE : burst sent / fifo_empty
    ST_FLUSH --> ST_IDLE : !trace_enable

4.5 死锁/活锁分析

  • 死锁防护 (Abstract Cmd): 1000 cycle 超时计数器, 超时后置 cmderr=010, 释放 busy
  • 死锁防护 (DMI): DMUI FSM 在 2 cycle 内完成 (PROC→DONE), DONE 等待 valid 撤除后回 IDLE
  • 死锁防护 (Trace FIFO): FIFO 满时丢弃 (overflow_count++), 不阻塞流水线
  • 活锁防护 (Halt): HALT_REQ 等待 commit_valid, 若流水线 stall 则 halt 延迟但不死锁 (stall 最终会被 flush 解除)
