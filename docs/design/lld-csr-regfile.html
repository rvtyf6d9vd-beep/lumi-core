4. 状态机定义

4.1 CSR 优先级仲裁逻辑 (隐式 FSM)

lumi_csr_unit 不含显式 FSM (无状态寄存器), 但 always_ff 中存在严格的 if-else if 优先级链, 等效于单周期隐式状态转移:

优先级条件动作等效状态
0 (最高)nmi_entermnepc/mncause/mnstatus 保存; mstatus 更新; 强制 M-modeNMI_ENTRY
1trap_enter && !trap_is_irqmepc/mcause/mtval/mstatus 更新; 切换到 M-modeEXC_ENTRY
2trap_enter && trap_is_irqmepc/mcause{1,…}/mstatus 更新; 切换到 M-modeIRQ_ENTRY
3mret_execmstatus MIE←MPIE; 特权级恢复到 MPPMRET
4sret_execmstatus SIE←SPIE; 特权级恢复到 SPPSRET
5mnret_execmstatus 从 mnstatus 恢复MNRET
6csr_we && csr_addr_valid && !is_clic_csr显式 CSR 写操作CSR_WRITE
7 (最低)默认 (每 cycle)mcycle/minstret/HPM 计数器递增HPM_INC

4.3 IRF 写冲突仲裁 (有限状态)

IRF 写冲突仲裁使用 1-entry pending buffer, 构成简单 FSM:

状态条件转移
IDLE (pend_vld=0)无冲突保持 IDLE; 正常双端口写入
IDLE → PENDirf_wr_conflict=1slot1 数据存入 pending buffer
PEND (pend_vld=1)port0 仍写同地址保持 PEND (等待 port0 释放)
PEND → DRAINport0 不冲突pending 数据通过 port1 grant 写入
DRAIN新 port1 写到达新写数据转为 pending (PEND)
DRAIN → IDLE无新 port1 写pend_vld 清零

4.4 死锁/活锁分析

  • 死锁防护: IRF pending buffer 仅 1 entry, 若连续多 cycle 双写冲突, pending 数据保持不丢, 新写覆盖旧 pending。不存在循环等待条件。
  • 活锁防护: 当 port0 连续写同一地址时 pending 无法 drain, 但此为极端罕见场景 (需连续多条指令写同一寄存器)。pending 数据保持正确, 一旦 port0 停止写该地址即可 drain。
  • CSR 仲裁: 单周期完成, 无多 cycle 状态, 不存在死锁/活锁。

BC-005NMI 与普通 Trap 同时到达优先级链: nmi_enter 最高优先级 (if-else if 第一项), NMI 处理, 普通 trap 被延迟到下一 cycleTC_NMI_TRAP_RACE
BC-006CLIC CSR 地址空间写: csr_addr = 0xFB0/0xFB1/0x347is_clic_csr = 1'b1; clic_csr_we = csr_we && is_clic_csr; CSR 单元不存储, 透传到 CLIC 模块TC_CLIC_PROXY
BC-007HPM 计数器 64-bit 溢出64-bit 自然回绕; hi_latch 每 cycle 从 cnt[63:32] 锁存, 软件读取 hi+lo+hi 三重校验TC_HPM_OVERFLOW
BC-008IRF 连续双写冲突: 多 cycle 持续 port0/port1 同地址pending buffer 保持旧数据不被覆盖; 新 slot1 写覆盖 pending; 一旦 port0 不冲突即 drainTC_IRF_CONFLICT_BURST
BC-009S-mode CSR 写 mstatus 子集: sstatus 写仅影响 SIE/SPIE/SPP仅更新 mstatus_r[1]/[5]/[8], 其余 M-mode 位保持不变TC_SSTATUS_SUBSET
BC-010复位后特权级priv_mode_r <= 2'b11 (M-mode); 全部 CSR 清零TC_RESET_PRIV

6.2 错误检测与恢复

错误类型检测方法恢复策略
非法 CSR 地址csr_addr_valid = 0 (default 分支)上游 Exception 模块检测并触发 illegal instruction trap (mcause=2)
IRF ECC 单 bit 错误ECC syndrome != 0 且奇校验ecc_ce_irq = 1; 软件可通过中断处理纠正
IRF ECC 双 bit 错误ECC syndrome != 0 且偶校验ecc_ded_irq = 1; 不可纠正, 触发 NMI 或系统停机
特权级违规S-mode CSR 在非 S/M-mode 访问由上游解码器检查, 本模块不执行特权级过滤 (由 HLD 设计决策)

6.3 未定义行为声明

  • csr_op = 2'b11: 未定义操作码。RTL 中 default: csr_write_val = csr_wdata, 行为等同 CSRRW。实际不应出现, 上游解码器应拦截。
  • NMI + MRET 同 cycle: NMI 优先级最高 (if-else if 链首项), MRET 被忽略。
  • CSR 写 mcause[31] (中断标志): 软件可通过 CSR 指令写任意值, 硬件仅在 trap 入口强制设置 bit[31]。

8.4 HPM 事件映射算法

// HPM 事件选择 (10-bit event ID → 事件信号)
for i in 0..HPM_COUNTERS-1:
    case mhpmevent_r[i]:
        0x000 → CPU_CYCLES    (每周期计数, hpm_event_match=1)
        0x001 → INST_RETIRED  (hpm_inst_retired)
        0x002 → BRANCH_MISS   (hpm_branch_mispred)
        0x003 → I_CACHE_MISS  (hpm_icache_miss)
        0x004 → D_CACHE_MISS  (hpm_dcache_miss)
        0x005 → EXCEPTION_TAKEN (固定 0, 由 trap 信号驱动)
        0x006 → IRQ_TAKEN     (hpm_interrupt)
        0x007 → CSR_ACCESS    (csr_we)
        default → 不计数 (hpm_event_match=0)

// 计数器递增 (每 cycle)
if hpm_event_match[i] && !hpmcounter_write_conflict[i]:
    mhpmcounter_r[i] += 1

// 写冲突检测
hpmcounter_write_conflict[i] = csr_we && csr_addr_valid &&
    (csr_addr == 0xB03+i || csr_addr == 0xB83+i)
